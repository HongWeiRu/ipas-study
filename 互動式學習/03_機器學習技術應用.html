機器學習基礎數學

機器學習不是魔法 — 它建立在三大數學支柱上：機率/統計（量化不確定性）、線性代數（表示與運算）、數值優化（找最佳解）。掌握這三者，AI 對你不再是黑盒子。

📌 三大支柱角色分工 · 不是「每個演算法都同時用三者」、而是「整個 ML 學科必須懂三者」
數學工具處理什麼問題核心應用演算法
機率 / 統計不確定性 + 推論樸素貝氏 Naive Bayes（純機率）、決策樹 Decision Tree（熵 / 吉尼）、邏輯迴歸（殘差分布）、所有評估指標（AUC, Area Under the Curve、信心區間 Confidence Interval）
線性代數向量 / 矩陣運算線性 / 邏輯迴歸（wx + b）、神經網路 NN / 深度學習 DL（每層矩陣乘法）、SVM, Support Vector Machine、PCA, Principal Component Analysis、K-NN、K-means

關鍵理解：不同演算法仰賴的工具不同。樹型演算法（決策樹、Random Forest、XGBoost）內部主要用機率 / 資訊論（Information Theory），幾乎不用線性代數；樸素貝氏純粹用機率乘法。但因為現代主流（神經網路、Transformer、大型語言模型 LLM）大量依賴矩陣運算（GPU / TPU 硬體就是為通用矩陣乘法 GEMM, General Matrix Multiply 設計），線性代數仍是 2026 最關鍵的支柱 — 這也是本章 §3.2 佔最多篇幅的原因。

機率 / 統計之機器學習基礎應用

真實世界的資料受限於觀察條件、取樣變異、內在隨機性 — 預測結果並非絕對唯一。機器學習的核心任務不是尋找絕對規則，而是處理「不確定性」並進行合理預測。這正是機率與統計成為 ML 三大數學支柱之一的原因。

本節主軸 · 把「不確定性」當成 ML 的第一原理 從 ChatGPT 每次生成下一個字（next-token softmax）、Stable Diffusion 反向去噪、Waymo 自駕車的 sensor fusion，到醫療 AI 必須附上信賴區間 — 2026 年的 AI 系統幾乎沒有「確定性答案」，只有「機率分佈與信心度」。掌握條件機率、貝氏定理、假設檢定，等於拿到讀懂現代 AI 內部運作的鑰匙。

A · 條件機率分佈 — ML 的核心邏輯

機器學習的核心邏輯：「在特定觀察條件下，某結果發生的可能性有多大」。模型並非尋求唯一解，而是學習一種條件機率分佈（Conditional Probability Distribution）。

P( Y | X )　　X = 輸入特徵（Feature）　Y = 目標變數（Label）　「|」讀作「在...條件下」

公式翻譯成中文：「在已知 X 的前提下，Y 各種可能取值的機率分別有多大」。X 是觀察值（影像像素、病歷數字、用戶歷史行為）、Y 是要預測的目標（圖片類別、是否會點擊、下一個字）。模型輸出的不是單一答案，而是整個 Y 的機率分佈 — 每個可能值都有對應機率，加總為 1。

垃圾郵件分類例：X = 郵件文字內容，Y ∈ {垃圾, 正常}。模型輸出 P(Y=垃圾 | X="這封郵件") = 0.92，表示在這封郵件文字條件下，判為垃圾的機率為 92%。

與傳統規則式 AI 的差異：1980 年代的專家系統（如 MYCIN）採 IF-THEN 邏輯，給「是/否」二元答案；1990 年代後的機率式 ML 給完整機率分佈 — 同一個病人，規則式回答「糖尿病」，機率式回答「正常 18% / 前期 25% / 糖尿病 52% / 其他 5%」，保留所有不確定性供下游決策（追蹤、加驗、轉診）。

傳統 IF-THEN vs 機率式 ML · 同一個診斷問題的兩種答案
同樣的病人 — 規則式給「YES/NO」（資訊量極小）；機率式給整個分佈（保留所有不確定性 → 醫生可決定再加驗、轉診、追蹤）
意涵說明
預測導向模型輸出為某結果的機率（如分類機率），而非確定性分類結果 — 給決策者完整訊息
不確定性評估機率反映模型對預測的信心程度，有助於風險控制與決策制定 — 例如：模型給「貓 51% / 狗 49%」與「貓 99% / 狗 1%」雖然都判為貓，但人類處理方式截然不同
📌 補充 · LLM 每一次「思考」都是條件機率（2025–2026） ChatGPT、Claude、Gemini 看似在「思考」，但本質只是在計算 P(下一個 token | 前文) — 一個 vocab size（通常 5–25 萬）維度的條件機率分佈。Temperature 參數 T 就是把 logits 除以 T 後再 softmax：T=0 取 argmax（接近決定性）、T=1 是原始機率、T>1 把分佈拉平讓低機率 token 也有機會。「創意」與「胡言亂語」的差別，只是 softmax 溫度 — 你大一機率課的 P(Y|X) 一路長大就成了 GPT-4。

B · 離散 vs 連續分佈

同樣是「隨機變數」，擲骰子跟量身高本質不一樣 — 骰子只可能落在 1、2、3、4、5、6 這六個位置上，是離散的；身高可以是 170.3 cm、170.31 cm、170.314 cm…一路細分下去永無止境，是連續的。這個差別不只是哲學區分 — 它決定了你如何寫下機率。

在離散的世界，「擲到 6」這件事有明確的機率 P(X=6) = 1/6，每個可能值都有對應的機率質量（mass） — 像把 1 公斤的米平均分到 6 個袋子裡。但在連續的世界，「身高恰好是 170.314159... cm」的機率是零（因為小數有無窮多位，要剛好命中那一點就是零機率）。所以我們不問「等於某個值」的機率，改問「落在某個區間」的機率 — 例如 P(170 ≤ 身高 ≤ 171) = 0.18。這就是 PMF（Probability Mass Function）與 PDF（Probability Density Function）的根本差別。

PMF vs PDF · 為什麼連續變數「單點機率 = 0」
離散：柱狀圖每根高度就是該值的機率 · 連續：機率 = 曲線下面積（單點寬度 = 0 → 機率 = 0）

三個必須認識的分佈 · 故事與直覺

常態分佈（Normal）之所以被稱為「正態」、又叫「鐘形曲線」，是因為它形狀像教堂的鐘：中間高、兩邊對稱、越遠機率越低。它無所不在 — 人類身高、體重、學測分數、製造業誤差、線性迴歸的殘差，都接近常態。背後有個深刻的數學理由叫中央極限定理：「大量獨立隨機因素加總起來，結果近似常態」。所以只要某現象是「多個小因素累加」的結果，幾乎必然呈現鐘形。常態分佈用兩個參數描述：μ（中心位置）與 σ（離散程度）— 後者越大鐘形越扁、資料越分散。

泊松分佈（Poisson）處理一種非常特定的場景：固定時間或空間區間中，某稀有事件發生的次數。客服中心每小時接到幾通電話、伺服器每分鐘收到幾筆請求、繁忙路口每天發生幾起車禍 — 這些都是泊松分佈的天然應用。它有個關鍵假設：事件之間互相獨立、平均發生率穩定（用 λ 表示）。如果今天平均每小時 10 通客服電話，明天突然湧入 50 通，那就不再是泊松了（可能是行銷活動或產品問題）。正因為「異常事件數」會偏離泊松，這個分佈也被用來偵測異常 — 例如網路流量檢測 DDoS 攻擊。

指數分佈（Exponential）專門描述事件之間的等待時間，而且有個迷人的性質叫「無記憶性」：手機已經用了 3 年沒壞，跟一支全新手機在「未來再撐 1 年」的機率是一樣的。這聽起來反直覺（畢竟舊手機應該更容易壞？），但對「純隨機故障」（如電子元件突發性損壞、原子衰變）這個性質是真的。指數分佈與泊松分佈是同一現象的兩面：如果某時段事件數服從泊松，那事件間的等待時間就服從指數。

C · 條件機率與貝氏定理

這一節要講的不只是兩條公式 — 而是一種看世界的方法。當新資訊出現，理性的人應該如何更新自己對世界的信念？兩百多年前的英國牧師 Thomas Bayes（1701–1761）給出了答案，但他生前並未發表，1763 年由朋友整理遺稿出版後沉寂了 200 年。直到 20 世紀後半，當電腦發展讓「反覆計算後驗機率」變得可行，貝氏定理才從哲學工具一躍成為現代 ML 的數學心臟。垃圾郵件分類、Waymo 自駕車定位、AlphaGo 蒙地卡羅樹搜尋、ChatGPT 的不確定性量化 — 通通建立在這條公式上。

① 條件機率 — 從「全集」縮小到「子集」

條件機率回答一個直觀問題：「當我知道 B 已經發生，A 發生的機率變成多少？」想像一個全班 30 人的教室，已知 18 人會打籃球（事件 A）、15 人會彈鋼琴（事件 B）、其中 6 人兩樣都會（A ∩ B）。如果隨機抽一個學生會打籃球的機率是 18/30 = 60%。但如果先告訴你「這個人會彈鋼琴」，問會打籃球的機率？這時參考框架從「全班 30 人」縮小到「會彈鋼琴的 15 人」，答案變成 6/15 = 40% — 這就是條件機率 P(A|B)。

P(A | B) = P(A ∩ B) / P(B)　　＝　兩者交集 ÷ 條件事件全集
條件機率 · Venn diagram 的「縮小視野」直覺
左邊看「全班 30 人中籃球比例」 → 60% · 右邊看「會彈鋼琴的 15 人中籃球比例」 → 40% — 條件機率 = 換視野

這個「縮小視野」的直覺，在 ML 中無處不在：

  • 分類任務中的條件預測：學習 P(Y|X) — 邏輯迴歸、貝氏分類器。把全世界的可能性 U，縮小到「觀察到輸入 X 的所有樣本」，再看 Y 的分佈
  • 生醫與金融風控：已知檢驗結果或行為模式 → 預估未來事件機率（罹病風險、違約機率）— 沒看到症狀前 vs 看到症狀後的機率天差地遠
  • 生成模型中的變數關聯：變分自編碼器（VAE）、隱馬可夫模型（HMM）建構潛在與觀察變數的依存 — 給定潛在狀態 z，觀察 x 的條件分佈

② 貝氏定理 — 從「現在」推回「原因」

條件機率 P(A|B) 跟 P(B|A) 是兩個不同的問題。前者問「看到 B，A 發生機率？」（症狀已知 → 推疾病）；後者問「已知 A，B 發生機率？」（疾病已知 → 推症狀出現率）。臨床數據通常給你後者（「有此病的人，95% 檢驗陽性」），但醫生真正需要前者（「檢驗陽性的人，真有此病的機率？」）。貝氏定理就是把後者反推回前者的橋樑。

P(A | B) = [ P(B | A) · P(A) ] / P(B)

Posterior		觀察到 B 之後，根據新資訊更新對 A 的重新估計 — 核心輸出
補充：貝氏推論在 ML 的關鍵價值 貝氏定理的關鍵意涵 — 把「原有知識（先驗）」與「新資料觀察（似然）」整合，產生即時的後驗機率調整。這種更新能力讓 ML 模型在面對不確定性與資料稀疏問題時保有推論彈性與解釋力。應用：垃圾郵件過濾（樸素貝氏）、推薦系統（貝氏個人化排序）、A/B 測試（貝氏 A/B Testing）。

貝氏的反直覺 · 99% 準確的檢驗為何仍可能 50% 是誤報？

這是統計學最著名的「陷阱題」 — 連醫生都常答錯。設定如下：

  • 某疾病的盛行率（先驗）：每 1,000 人中有 10 人罹病 → P(病) = 1%
  • 檢驗試劑準確率：真有病的人，99% 會驗出陽性 → P(陽性|病) = 99%
  • 同樣 99% 準確率：沒病的人，99% 會驗出陰性（也就是 1% 偽陽性）→ P(陽性|沒病) = 1%

問題：你拿到陽性結果，真有病的機率多少？很多人會脫口而出「99%！」 — 大錯特錯。讓我們把 1,000 人攤開來算：

貝氏反直覺 · 1,000 人視覺化計算
不要被「99% 準確率」誤導 — 先驗（盛行率）+ 似然（檢驗率）+ 邊際 = 後驗。三者缺一不可

這個例子之所以重要，不只因為它反直覺、更因為它教會我們看 ML 模型輸出的正確方式：

  • 不要只看「準確率」 — 在類別不平衡資料（如詐騙交易、罕見病、設備故障）上，準確率高的模型可能完全沒用
  • 要看 Precision / Recall / F1 — 都是條件機率的不同切法（後面 §5.3 模型評估會深入）
  • 先驗很重要 — 同一個模型在不同盛行率場景下表現會差很多；訓練資料與部署環境若先驗不同，要做類別重新加權或校準（calibration）
貝氏定理 · 三步走的「信念更新」
貝氏定理 = 把「原有信念」用「新證據」更新成「新信念」 — 推薦系統、自駕車、垃圾郵件分類器都靠這條公式

真實案例 · 樸素貝氏（Naive Bayes）垃圾郵件分類

CLASSIC ML APPLICATION

背景：1998 年 Sahami 等人首次將貝氏定理用於垃圾郵件過濾，至今仍是 Gmail / Outlook 過濾基線之一。

數學公式（給定郵件詞袋 w₁..wₙ，判斷是否為垃圾郵件 S）：

P(S | w₁..wₙ) ∝ P(S) × ∏ P(wᵢ | S)

具體流程：

  • 先驗 P(S)：歷史所有郵件中，垃圾郵件比例（如 30%）
  • 似然 P(wᵢ|S)：「免費」這個字在垃圾郵件中出現的機率（如 80%），在正常郵件中出現的機率（如 5%）
  • 後驗 P(S|郵件)：看到這封含「免費」的郵件後，它是垃圾的機率劇增 → 過濾
  • 「Naive」的原因：假設每個字詞獨立 — 不真但夠用，計算量從 O(2ⁿ) 變 O(n)

為何此案例經典：具體展示「先驗 × 似然 → 後驗」如何用幾行公式做出商業價值；任何想真正理解貝氏定理的人都該手算過一次。

📌 補充 · Waymo 自駕車：貝氏定理在 100 毫秒一次的工業實踐 Waymo Robotaxi 上路時，每 100 毫秒就會用 Extended Kalman Filter 把 29 顆相機、5 顆 LiDAR 與多顆 Radar 的觀測值融合成「車輛在世界中位置」的後驗機率分佈。本質就是貝氏定理的遞迴版本：先驗（上一刻的位置估計）× 似然（感測器讀數）→ 後驗（這一刻的位置估計）。2025 年實測高速公路 EKF 狀態估計誤差 ±0.3 m、100 m 內定位誤差 < 0.15 m。學會貝氏定理，你就理解了所有自駕車背後的數學主幹。

D · 統計推論與假設檢定

想像你煮了一大鍋湯，要判斷夠不夠鹹。你不會把整鍋喝光 — 你嚐一口就決定。但這一口能不能代表整鍋？如果攪拌不均勻，可能你正好嚐到鹽巴沉底那塊。「樣本能否代表母體」就是統計推論的核心焦慮。

ML 工程師面對的是同樣困境：你想知道「全世界所有用戶對新版產品的反應」（母體），但你永遠不可能測到全部 — 你只有幾千筆 A/B 測試資料（樣本）。從有限樣本推論母體真相，就是統計推論在做的事。這套方法有兩個分支：① 參數估計（猜母體的值是多少）；② 假設檢定（驗證某個主張是否站得住腳）。

母體 vs 樣本 · 統計推論的根本鴻溝
統計推論 = 用 樣本（紫色點，可觀測）推測母體（黑色點，不可觀測）的真實參數
面向參數估計假設檢定
目的推測母體參數的「值」或「區間」驗證某主張是否成立
核心問題母體參數大約是多少？是否有足夠證據拒絕一個假設？
輸出點估計值 + 信賴區間（如 95% CI）p 值、檢定統計量 + 顯著水準決策
例子樣本平均體重 68kg，95% CI 為 [66, 70]新運動課程降低體重，p = 0.03 < α = 0.05，拒絕 H₀，認為有效

信賴區間（CI）到底是什麼意思？

「95% 信賴區間 [66, 70]」這句話被大量誤解。它不是說「真平均有 95% 機率落在 [66, 70]」 — 真平均要嘛在這區間裡、要嘛不在，只是個固定數字而已。正確解讀是：「如果我用同樣方法抽樣 100 次、算 100 個 CI，那這 100 個區間中大約有 95 個會包含真平均」。CI 的隨機性來自「抽樣這個動作」，不是來自真平均本身。

95% 信賴區間 · 100 次模擬抽樣的視覺真相
每條橫線是一次抽樣的 95% CI · 紅色虛線是真值 μ。長期重複下，約 95% 的 CI 會「打中」真值

假設檢定五步驟流程

1
STEP 01

設定虛無假設 H₀ 與對立假設 H₁

H₀ 為現狀（無差異、無效果）；H₁ 為研究者想驗證的新主張。

2
STEP 02

選擇適當的檢定方法與檢定統計量

視資料型態與比較對象選擇（t 檢定、卡方、ANOVA、Z 檢定）。

3
STEP 03

決定顯著水準 α

α = 0.05 是慣例；醫療等嚴謹領域用 0.01。

4
STEP 04

計算檢定統計量與 p 值

p 值含義：在 H₀ 為真的前提下觀察到目前樣本（或更極端）的機率。p 越小，越支持拒絕 H₀。

5
STEP 05

比較 p 值與 α 做決策

p < α → 拒絕 H₀；p ≥ α → 無法拒絕（不等於接受）。

α vs p 值常被混淆 α 是事前設定的判斷門檻；p 值是事後計算的觀察結果。在 ML 中，假設檢定常用於特徵選擇（卡方檢定篩選對目標變數顯著的類別特徵）、模型比較（兩模型表現差異是否統計顯著）、A/B 測試（新版本是否真的優於舊版本）。

兩種出錯的方式 · 型一錯誤 vs 型二錯誤

假設檢定有兩種出錯方式 — 而且降低其中一種會放大另一種，沒有萬靈丹。「型一錯誤」（False Positive）是「假警報」：事實上沒問題，卻被你誤判為有問題（H₀ 為真，你卻拒絕）；「型二錯誤」（False Negative）是「漏報」：事實上有問題，卻被你誤判為沒問題（H₁ 為真，你卻沒拒絕 H₀）。顯著水準 α 就是「容許型一錯誤的上限」 — α=0.05 = 「我願意承擔最多 5% 機率把無辜誤判為有罪」。

混淆矩陣 · 假設檢定 vs 真實情況的四個格子
α（型一）= 沒問題卻被誤判 · β（型二）= 有問題卻沒抓到 — 降一個就放大另一個，無法雙贏
p 值 vs α · 顯著水準的視覺意涵
α 是事前設定的「容忍犯錯機率」；p 值是事後計算的「實際觀察極端度」 — 兩者比大小做決策

真實案例 · Netflix 序列式假設檢定（2024 升級版 A/B）

SEQUENTIAL TESTING · ABLAZE

背景：Netflix 自家實驗平台 ABlaze 每天同時跑數百個 A/B test（3 億+ 用戶基數）。傳統 A/B testing 必須事先固定樣本數，否則「中途偷看」會破壞型一錯誤保證 — 但業務團隊就是想隨時看結果。

2024 升級：Netflix Tech Blog 2024 年 2 月起發表「any-time valid sequential testing」與「Bayesian 序列檢定」系列，運用混合鞅（mixture martingale）構造允許隨時偷看仍保證型一錯誤 ≤ α 的 confidence sequences。

對教科書的衝擊：「固定 n 才能算 p-value」這條規則被改寫；現代大規模 ML 系統的假設檢定必須升級為 anytime-valid 版本才能跟上產品節奏。Bayesian A/B test 則直接輸出「B 比 A 好的後驗機率」，更貼近商業決策語言。

真實案例 · Spotify Contextual Bandits + Bayesian Personalized Ranking

BANDITS · BPR · 2025

背景：Spotify 2025 Q3 月活躍用戶 7.13 億。傳統 A/B 在實驗期間會把流量「浪費」在較差的選項上；contextual bandit 用「探索/利用」自動分流，邊學邊賺。

2025 部署：Spotify Research 2025 年 9 月公開「Calibrated Recommendations with Contextual Bandits on Spotify Homepage」 — 首頁的歌單卡片排序，背後是貝氏後驗分佈在每次刷新時被更新；推薦演算法骨架使用 Bayesian Personalized Ranking（BPR）。

另一例 · Dream11（印度幻想體育龍頭）：2025 年 5 月上線 hierarchical contextual uplift bandits 作為預設個人化系統，相較傳統 A/B testing 帶來 0.5% 營收提升 — 對 7 億用戶級平台是上千萬美元差距。

教學意義：學會 A/B test 不夠，要懂「線上學習」 — 條件機率 P(喜歡 | 用戶情境) 在工業界已從「離線估計」變成「即時更新」。

📌 補充 · 因果推論（Causal Inference）正在取代純相關性 ML 傳統 ML 學的是「相關性」，但業界要回答的是「如果我降價 10%，訂單會多多少？」— 這是因果問題。Uber 2024 年發表「Practical Marketplace Optimization Using Causally-Informed ML」（arXiv 2407.19078），結合深度網路、凸優化、形狀受限模型估計補貼/定價/配對的真實效應；KDD 2024 首次設立 Causal Inference Workshop。未來 ML 工程師必須懂 Pearl 因果圖、Potential Outcomes、Double ML — 「會做 A/B test」已經不夠。

E · 統計量在 ML 中的角色

在動手訓練模型之前，資料科學家會做一件事 — 看資料的形狀。形狀不對，後面所有模型都會出問題。「形狀」用四個統計量描述：① 期望值（平均）說中心在哪；② 變異數/標準差說資料多分散；③ 偏度說歪向哪邊；④ 峰度說尾巴多厚。這四個數字決定你需不需要標準化、要不要做對數轉換、是否該移除離群值、能否套用常態分佈假設。

平均數會騙人 · 為什麼還要看中位數、偏度、峰度

經濟新聞常說「台灣家庭年收入平均 120 萬」 — 多數人聽完一頭霧水，因為自己跟周圍人都遠低於這個數字。原因：少數極高收入家庭把平均拉高了。台灣家庭中位數收入其實只有約 88 萬 — 比平均低了 32 萬。這就是「右偏分佈」的典型現象：尾巴拖到右邊，平均數被尾巴拉高，但大多數人位於中位數附近。看資料只看平均，就像只看群體最高薪資來判斷一份工作好不好。

偏度與峰度 · 同樣是「鐘形」但形狀差很大
偏度 = 歪向哪邊（左偏/對稱/右偏）· 峰度 = 尾巴厚度（金融市場的「黑天鵝」就是高峰度）

期望值
Expected Value		反映變數平均趨勢；許多模型的預測基準與參數估計核心（線性迴歸截距項）
變異數 / 標準差
Variance / SD		衡量資料離散程度；判斷特徵是否需標準化，避免尺度不一致對訓練造成偏誤
偏度 Skewness判斷分佈是否對稱；偏態過大常需對數轉換或 Box-Cox 轉換改善模型收斂

實務口訣 · 看一份新資料的標準動作：① 畫 histogram 看形狀（鐘形？偏？多峰？）→ ② 畫 boxplot 找離群值 → ③ 算 偏度判斷要不要轉換 → ④ 算 峰度判斷要不要 robust loss → ⑤ 畫 QQ-plot 確認是否近似常態。這五步驟做完才動模型 — 否則你訓練的是「想像中的資料」而非「真實資料」。

常考點：機率分佈與 ML 模型對應 題目情境問「邏輯迴歸假設目標變數服從什麼分佈？」 → 答伯努利分佈。問「線性迴歸的誤差項假設？」 → 答常態分佈。問「客服每小時來電次數模擬？」 → 答泊松分佈。記住：選分佈 = 選模型的先驗假設。
📌 補充 · 醫療 AI 與 FDA：信賴區間從「學術名詞」變成「法規必備」 自 2024 年起，美國 FDA 強制要求所有 AI 醫療器材送審必須附效能指標的信賴區間（CI），不能只報「準確率 95%」。原因：模型在 1,000 張影像上達到 95% 與在 100 萬張影像上達到 95%，臨床可信度完全不同。2025 年 RSNA Radiology「AI Predictive Uncertainty: A Step Forward」社論進一步推動 aleatoric（資料噪音）與 epistemic（模型知識）雙重不確定性報告。AI 應用規劃師若不懂統計推論與 CI，就無法判斷供應商提供的模型是否可上線。
📌 補充 · LLM 幻覺的統計學解方：Uncertainty Quantification Heads（2025） 2025 年 arXiv 論文「A Head to Predict and a Head to Question」（2505.08200）為 Mistral、Llama、Gemma 2 預訓練 UQ heads，可在生成時同步輸出 token 不確定性分數，用於 hallucination 偵測。同年 12 月 arXiv 2512.19920「Behavior-Calibrated RL」進一步驗證：一個 4B 小模型經 calibrated RL 訓練後，校準誤差可媲美 Grok-4、Gemini-2.5-Pro。「會說我不知道」是比「答案正確」更高階的能力 — 也是「機率/統計」這一節對工程師最直接的應用。
📌 補充 · 貝氏深度學習與 Meta Ax 1.0（2025/11） Bayesian Neural Networks（BNN）把網路權重視為機率分佈而非固定數值，能天然輸出預測不確定性。2025 ICLR 收錄論文「Streamlining Prediction in Bayesian Deep Learning」推進可微分邊際似然估計；同期理論研究亦指出 mean-field VI 在大寬度網路會退化成先驗預測。應用面：Meta 於 2025-11-18 開源 Ax 1.0，使用 Bayesian Optimization + Gaussian Process 為 AI 模型/基礎設施/硬體設計自動化超參數實驗，比 grid / random search 省 10–100× 計算成本。對標 Optuna、Google Vizier、AWS SyneTune — 這是「機率/統計」在 LLMOps 時代最具經濟價值的延伸。
§3.1 小結 · 一頁讀懂機率/統計如何撐起 ML ① ML 的核心 = 學 P(Y|X) 條件機率分佈（不是學唯一答案）；② 選分佈 = 選模型的先驗假設（伯努利/常態/泊松對應邏輯迴歸/線性迴歸/事件計數）；③ 貝氏定理 = 先驗 × 似然 → 後驗（垃圾郵件、Waymo Kalman、推薦系統全靠這條）；④ 假設檢定 = α 事前 / p 值事後（Netflix 升級為 sequential testing、Spotify 改用 contextual bandit）；⑤ 統計量是資料探索與特徵工程的眼睛（標準化、轉換、異常偵測）；⑥ 2025–2026 趨勢：UQ heads、FDA CI、因果推論、Bayesian Optimization全在強化「不確定性管理」這條主軸。

線性代數之機器學習基礎應用

線性代數是 ML 模型運算與表示的數學基礎。絕大多數演算法都以矩陣與向量為運算單位，透過線性變換、特徵分解、最小平方估計實現模型訓練與預測。從資料表示、參數計算、梯度更新到特徵轉換 — 每個環節都離不開線性代數。

本節主軸 · 為什麼說「整個 LLM 革命是線性代數的勝利」 GPT-4、Claude、Gemini 內部 99% 算力都花在矩陣乘法（GEMM）上；NVIDIA H100 / B200 賣那麼貴是因為 Tensor Cores 為 GEMM 而生；LoRA 讓微調百億參數模型可以人人玩，依靠的是低秩近似；RAG 系統的「相似搜尋」就是向量內積；JPEG 影像壓縮是換基底；Google PageRank 是主特徵向量。掌握線性代數，等於拿到讀懂現代 AI 軟硬體棧的鑰匙。

A · 向量與矩陣的角色

電腦看不懂「一個 35 歲、住信義區、月薪 8 萬、喜歡爵士樂的男性」這句話 — 但它看得懂 [35, 1, 80000, 0.85, 1] 這串數字。當你把一個人的所有特徵排成一串數字，這個「一串數字」就是向量。整批人攤開來，每人一列，就是矩陣。從這一刻起，「用 ML 處理問題」就等同於「對向量與矩陣做數學運算」 — 線性代數因此成為機器學習的母語。

② 矩陣 — 多筆樣本或權重的集合

矩陣意義
特徵矩陣 X ∈ ℝⁿˣᵈn 筆樣本、每筆含 d 個特徵；每一列為一筆樣本向量

三大核心矩陣運算

  • 轉置（Transpose）：列與行互換；維度對齊與內積計算
  • 矩陣求逆 / 偽逆（Inverse / Pseudo-Inverse）：用於封閉解（最小平方解），或無法反矩陣時近似

B · 線性變換 — 幾何意義

想像你拍了一張照片，然後做幾個動作：① 放大兩倍（縮放）；② 順時針轉 30°（旋轉）；③ 把它拉斜成平行四邊形（剪切）；④ 壓扁成一條直線（投影）。這四種操作的共同點是：原本的「直線」依然是直線、「原點」依然是原點 — 沒有把直線扭成曲線、沒有把空間撕裂。所有滿足這兩個條件的操作，都叫「線性變換」，而且都可以用一個矩陣 A 來表達。

這個觀點非常重要 — 因為神經網路裡每一層的運算，本質上就是一次線性變換 + 一次非線性激活。當你看到「784 維輸入 → 128 維隱藏層 → 10 維輸出」的網路，等於是把 784 維空間「變形」到 128 維、再「變形」到 10 維。Embedding 層、Attention 機制、PCA 降維 — 全部都是「對空間動手腳」的線性變換。學會用幾何想像力看矩陣運算，深度學習就不再是黑盒子。

當向量 x 被矩陣 A 左乘 → 新向量 Ax 是對原向量的一次線性變換。可能在同維度中發生，也可能投射至另一維度的空間。

GEOM 01
縮放 Scaling
調整向量在各方向的長度，改變尺度但不改變方向
GEOM 02
旋轉 Rotation
改變方向但不改變長度；常見於正交變換或特徵對齊
GEOM 03
剪切 Shearing
向量方向在空間中產生傾斜變化；非對角矩陣的變換
GEOM 04
投影 Projection
將高維向量投射到子空間（主成分空間或分類超平面）；保留對任務最有意義的資訊

C · 矩陣分解 — 降維與特徵抽取

想像你有一張 1080×1920 的照片 — 表面上是 2 百萬個像素，但實際上一張臉孔的「本質特徵」可能用 200 個數字就描述完了（兩眼間距、鼻翼寬度、下巴角度…）。同樣的，Netflix 5 億用戶 × 10 萬部電影的評分矩陣表面上是 50 兆個格子，但用戶偏好其實只在幾十個維度上變化（喜歡動作片？愛看老片？需要中文配音？）。矩陣分解就是這個觀察的數學版本：原始矩陣表面又大又複雜，本質上由少數幾個「基底因子」組合而成 — 找到這些因子，就能壓縮、降維、推薦、生成。

用一個比喻：原始矩陣像是「最終菜色」，分解後的子矩陣像是「食材 + 配方」。同樣一道番茄炒蛋，你看到的是「炒好的菜」（觀察值矩陣），但廚師需要的是「番茄+蛋+油+鹽 ×（炒這個動作）」（基底因子）。掌握食材就能複製、變形、簡化菜色 — 這就是矩陣分解的力量。

矩陣分解的「降維本質」· 為什麼大矩陣 ≈ 小矩陣相乘
同一張 Netflix 評分矩陣 — 直接存 50 兆格 vs 分解成兩個 50 維因子相乘只要 250 億格

矩陣分解（Matrix Factorization）把高維矩陣拆解為多個較小且具結構意義的子矩陣 — 實現降維、壓縮、轉換。下面三種分解（EVD / SVD / NMF）是 ML 領域最常見的工具，差別在於「什麼樣的原矩陣可以這樣分」與「分出來的因子有什麼物理意義」。

① 特徵值分解 Eigenvalue Decomposition

EVD

原理：適用於對稱方陣；將矩陣分解為特徵向量與特徵值的組合。

A = Q Λ Qᵀ

其中 Q 是正交矩陣（由 A 的特徵向量構成）、Λ 是對角矩陣（對角元素為特徵值）。

幾何意義：找出一組能穩定表示資料「拉伸方向」的基底，並量化每個方向的重要性。

應用：① PCA 主成分分析把資料投影到最大變異方向；② LDA 線性判別分析找最佳分類投影方向。

② 奇異值分解 SVD

Singular Value Decomposition

原理：可應用於任意實數矩陣（不需為方陣），分解為三部分：

X = U Σ Vᵀ
  • U ∈ ℝᵐˣᵐ：左奇異向量矩陣（對應樣本方向）
  • Σ ∈ ℝᵐˣⁿ：奇異值對角矩陣（對角線為非負實數，表示主方向重要性）
  • V ∈ ℝⁿˣⁿ：右奇異向量矩陣（對應特徵方向）

應用：

  • 資料降維：保留前 k 個奇異值與向量，近似原始資料（PCA 計算）
  • 推薦系統：分解使用者-項目矩陣，找潛在偏好向量
  • 潛在語意分析 LSA：抽取語料中詞與文件間的潛在語意
  • 影像壓縮：只保留主成分影像資訊

③ 非負矩陣分解 NMF

Non-negative Matrix Factorization

原理：把非負矩陣 X ≥ 0 分解為兩個非負矩陣乘積：

X ≈ W H　（W, H ≥ 0）

幾何意義：把資料視為幾個「可加疊的部件」 — 具備語意解釋力，並自然引入稀疏性。

應用：主題建模（Topic Modeling，將文件-詞矩陣分解為主題與詞彙分佈）、生物訊號分析（腦波分解）、影像分析（視覺元素拆解）、社群分析（互動關聯）。

補充：三種分解的差別記憶 EVD（特徵值分解）：只能用於對稱方陣；給你「主要方向」
SVD（奇異值分解）：任意矩陣都能分解；給你「樣本方向 + 特徵方向 + 重要性」 — 最通用
NMF（非負矩陣分解）：限制非負；給你「可加疊的部件」 — 解釋性最強
實務上 PCA 多用 SVD 實作（數值穩定、不需先計算協方差矩陣）。

真實案例 · Netflix Prize — SVD 矩陣分解的百萬美元競賽

2006–2009 · $1M COMPETITION

背景：2006 年 Netflix 懸賞 100 萬美元，徵求提升推薦準確度 10% 的演算法 — 引爆了矩陣分解的黃金年代。

Simon Funk 的 SVD-style 矩陣分解：把 50 萬用戶 × 1.7 萬部電影的稀疏評分矩陣（稀疏度 > 99%）拆解成兩個 50–200 維的潛在因子矩陣相乘：R ≈ U · Vᵀ，每個用戶被表達為一個 latent vector、每部電影也是一個 latent vector，相似的人 / 電影在向量空間靠在一起。

R[用戶×電影] ≈ U[用戶×k] · V[k×電影]　　k = 50 ~ 200

結果：Funk 演算法拿下第三名，被廣為複製；冠軍 BellKor's Pragmatic Chaos 也是 107 個演算法（含矩陣分解）的 ensemble。

2026 仍然有效：Spotify、Amazon、YouTube 至今仍在用矩陣分解作為推薦系統的基礎模組 — 線性代數的低秩近似比任何深度學習都早 10 年支配商業 ML。

真實案例 · Google PageRank — 矩陣特徵向量的 2 兆美元應用

EIGENVECTOR · POWER ITERATION

背景：1998 年 Larry Page 與 Sergey Brin 在 Stanford 提出 PageRank — Google 一整套搜尋帝國的數學起點。

核心想法：把每個網頁的連結關係寫成列隨機矩陣 A（每個網頁的「票數」按出連結平均分給其指向的網頁），網頁重要性 = A 的主特徵向量（對應特徵值 λ = 1）。

A · v = v　　（即 λ = 1 的特徵向量）

計算方式：用 power iteration 反覆計算 v ← A·v 至收斂即得排名。PageRank 矩陣維度 = 全網頁數（兆級），但 power iteration 通常只需 50–100 次迭代收斂。

教學意義：Larry Page 和 Sergey Brin 用 Ax = λx 這條大一線性代數公式，建立了一家市值兩兆美元的公司 — 「會 SVD」不是學術擺設，是商業 ML 起點。

📌 補充 · LoRA — 低秩近似讓 LLM Fine-tuning 從 GB 降到 MB LoRA（Low-Rank Adaptation, Microsoft 2021）是線性代數低秩近似的明星應用：與其更新完整的 d×d 權重矩陣（百萬參數），不如假設更新的「資訊維度」很低，用兩個小矩陣 A（d×r）和 B（r×d）相乘來逼近殘差更新 ΔW = B·A（r 取 8 即可）。對 GPT-3 175B，LoRA 把可訓練參數降低 10,000 倍、GPU 記憶體降 3 倍；rank=8 配置下 32 層僅約 420 萬參數（8MB fp16），壓縮逾 1000 倍。這就是為什麼 Hugging Face 上有上萬個社群 fine-tune 模型 — 「人人玩得起 LLM 客製化」的數學原因。
📌 補充 · INT4 量化讓 Llama 3.1 405B 從 2 台 server 降到 1 台 4 卡 量化（Quantization）把模型權重從 FP16（16 bit）壓到 INT8/INT4，運用低秩 + 範圍縮放技術維持精度。Meta 為 Llama 3.1 405B 推出官方 FP8 量化版（覆蓋 75% inference FLOPs）；Neural Magic 把 405B 量化到 INT4 仍保持約 100% 準確度。VRAM 需求從 ~810 GiB 降到 ~203 GiB，從 2 台 8×80GB GPU server 變成 1 台 4×H100 即可跑，成本降約 4 倍。背後核心是「找一組縮放因子，讓壓縮後的整數向量仍能逼近原始浮點向量」— 線性代數量化的工業勝利。
📌 補充 · JPEG / HEIF 影像壓縮的線性代數本質 每次你拍照存成 JPG，手機都在做大量線性代數 — JPEG 把影像切成 8×8 小塊，對每塊執行 2D 離散餘弦變換（DCT），把 64 個像素值轉換成 64 個正交餘弦基底的線性組合。因為人眼對高頻細節不敏感，量化掉高頻係數就能達到 10:1 壓縮。HEIF / WebP / AV1 用類似的整數變換。這正是「換一組基底，讓資訊集中在少數座標軸」的線性代數哲學 — 與 PCA 同一個思想脈絡。
📌 補充 · 雙塔模型（Two-Tower）— 現代推薦系統的線性代數通用解 Spotify / Pinterest / TikTok 的推薦引擎都收斂到同一線性代數模式：雙塔模型。一個 tower 把用戶編碼成向量、另一個 tower 把內容（音樂、Pin、短影片）編碼成向量，兩者在同一個 embedding 空間做內積匹配，最相關的內容就是「向量夾角最小」那些。Spotify 2T-HGNN 為 5 億用戶推薦音樂與有聲書；Pinterest PinSage 把這套用到 30 億張 Pin，是史上最大規模的圖嵌入應用。

D · 最小平方估計與線性迴歸

假設你有 30 筆資料：30 戶房子的坪數（輸入 x）與售價（輸出 y），畫在散布圖上。直覺上，坪數越大房價越高 — 但這些點不會剛好排成一條直線。線性迴歸要回答的問題是：「畫哪一條直線最能代表這 30 個點？」每條候選直線會跟 30 個點分別產生「誤差」（點到直線的垂直距離），誤差總和最小的那條，就是最佳擬合線。這是線性迴歸 200 年來的核心思想 — 由 19 世紀數學家 Carl Friedrich Gauss 在 1809 年研究天文軌道時系統化。

但這裡有個關鍵設計選擇：「誤差總和」到底怎麼算？有人說「正負抵銷會出問題 → 取絕對值」（MAE）；有人說「絕對值不可微分、不好計算最佳解 → 取平方」（MSE）。「平方」這個選擇就是 OLS（Ordinary Least Squares，最小平方法）名字的由來。平方有兩個好處：① 永遠正、不會抵銷；② 處處可微分、有解析解（一條公式直接算出最佳參數）。代價是：對極端值非常敏感（誤差 10 → 平方 100；誤差 100 → 平方 10000，差距被放大 100 倍）— 後面 §3.3 損失函數會深入這個權衡。

最小平方法 · 房價散布圖找「最佳直線」
同樣 10 個點 — 直線 A 誤差大、直線 B 誤差最小。OLS 用「平方誤差總和最小化」找到唯一最佳線

最小平方估計 OLS

Ordinary Least Squares

核心概念：以「誤差最小化」為目標的參數估計方法。比較模型預測值與實際觀測值的差異，找出一組參數讓差異平方總和達到最小。

幾何觀點：線性迴歸的本質是一種投影 — 把輸入資料在特徵空間中投影到最接近實際結果的平面上。這個平面就是模型學到的線性關係。

侷限：資料若分佈過於分散或存在離群點，擬合平面可能會被嚴重扭曲（一個遠離大群的點被「平方」放大後會主宰整條線的方向）。

A · 最佳化問題的四大構成

想像你被矇上眼睛丟進阿爾卑斯山的某座山谷裡，任務是「找到整座山谷的最低點」。你做不到「飛到上空俯瞰全局」 — 只能用腳感受地面斜度、選一個下坡方向、走一步、再感受、再走。這就是訓練神經網路的全部。

在這個比喻裡：① 你的高度 = 損失函數（loss）值 → 越低越好；② 你的位置座標 = 模型的所有可訓練參數（神經網路裡通常數百萬到數千億個）；③ 地面的斜度 = 損失函數對參數的梯度；④ 每步的大小 = 學習率（learning rate）。整個訓練過程，就是沿著梯度方向反覆走小步、希望走到山谷底部。Llama 3.1 405B 訓練 54 天，就是在 4,050 億維空間裡走了無數小步 — 規模驚人，但本質就是這個比喻。

把這個直覺整理成數學語言，所有的最佳化問題（不限於 ML）都有四大構成要素。下表四項就是「把一個現實問題翻譯成可解的最佳化問題」必須先想清楚的：

② 決策變數

B · 損失函數設計

山谷比喻裡的「高度」是什麼？這就是損失函數（Loss Function）的工作 — 把「模型預測得多差」翻譯成一個可比較的數字。但「翻譯方式」的選擇，會徹底改變學習行為。同樣是「房價預測誤差 5 萬」，有人覺得跟「誤差 1 萬」差距「5 倍」、有人覺得是「25 倍」 — 不同的觀點對應不同的損失函數，最後訓練出來的模型行為截然不同。

最經典的對比是 MSE（均方誤差）vs MAE（平均絕對誤差）。MSE 把誤差平方後再平均，MAE 取絕對值後再平均。差別看起來小，效果差很大 — 因為「平方」會放大大誤差的影響：誤差 10 → 平方變 100；誤差 100 → 平方變 10,000，放大 100 倍。這讓 MSE 對極端值極度敏感 — 一個離群房子（豪宅）可能會主宰整條迴歸線的方向。

MSE vs MAE · 同樣 10 筆資料 + 1 個極端值，兩種損失的反應
MSE 把大誤差「平方放大」→ 一個豪宅可主宰整條線 · MAE 對極端值穩健，但不可微分 → Huber Loss 是兩者折衷

損失函數不只是誤差量化工具，更體現學習目標的策略偏好與風險容忍度。設計決定模型如何看待不同型態的錯誤、哪些誤差應被放大懲罰、參數調整的方向與幅度。

C · 優化演算法 — 梯度下降系列

回到山谷比喻：你知道要「沿斜度下坡」，但每一步要看多少資料才決定方向？這就是 GD（梯度下降）家族最根本的權衡。

GD（全批次梯度下降）就像一個謹慎的登山者：每走一步前，先把整個山谷地形都測量一遍，計算精確的平均斜度後才邁出。優點是方向永遠正確，缺點是慢得受不了 — 訓練資料若有 1 億筆，每走一步要看完全部 1 億筆才能更新。SGD（隨機梯度下降）是另一極端：「反正全測太慢、那我隨便抓 1 筆資料就走」。速度飛快，但方向極度震盪 — 可能東走一步、西走一步，看起來像喝醉的人下山。Mini-batch SGD是兩者的折衷：每次抓 32 / 64 / 256 筆小批，方向夠穩、速度也快 — 完美契合 GPU 的並行運算（GPU 一次跑 256 筆遠比跑 1 筆有效率）。這就是 2026 年深度學習標準做法。

基本梯度下降會遇到三個實務挑戰：① 梯度方向不穩定（震盪、鞍點）；② 參數更新速度不一致；③ 固定學習率難應對不同訓練階段。進階優化器解決這些問題。

理解這些優化器最快的方法是小球滾下山的比喻：① SGD 是一顆沒有質量的小球，每一步只看當下斜度走 — 遇到平坦處停滯，遇到鞍點容易卡住；② Momentum 是給小球加質量、加慣性 — 即使遇到平坦處還是會憑慣性繼續滾，越過小山丘也不會立刻停；③ Adagrad / RMSprop 是給小球裝智慧煞車，常常在改變方向的維度上自動減速、平穩的方向上維持速度；④ Adam 就是「有慣性 + 有智慧煞車」的綜合體 — 這就是為什麼它成為 2014 之後深度學習的預設選擇。後面 §5.4 模型調整與優化會深入這些技巧。

Momentum 動量法

MOMENTUM

原理：模仿物理動量，把前幾次梯度方向累積，幫助模型克服局部震盪。

優點：加速收斂、穩定學習過程，特別適合高曲率或非平滑空間。

應用：深層神經網路訓練、收斂速度要求高的任務。

Adagrad

ADAPTIVE GRADIENT

原理：根據每個參數的歷史梯度大小，自動調整學習率。

優點：適合處理稀疏特徵或不均衡參數更新；對變動少的參數保留高學習率、變動大的降低。

限制：學習率會隨時間過度衰減，可能導致收斂停止。

RMSprop

ROOT MEAN SQUARE PROP

原理：引入滑動平均，修正 Adagrad 學習率過快下降的問題。

優點：能穩定訓練，特別適合處理非穩定梯度（RNN 訓練）。

應用：語音處理、序列建模、非凸問題。

Adam — Adaptive Moment Estimation

當前最常用

原理：結合 Momentum + RMSprop，追蹤梯度的一階與二階動量，自動調整各參數的學習率。

優點：訓練快速、收斂穩定、可廣泛應用於大多數模型。

應用：目前最常用的深度學習優化器，適用圖像、語言、強化式學習等各類任務。

補充：優化器選擇邏輯 實務上：① 大多數情境用 Adam（預設值 lr=0.001 通常夠用）；② 對 RNN 用 RMSprop；③ 對極端稀疏特徵用 Adagrad；④ 想精細調控用 SGD + Momentum + LR Schedule（效果有時超過 Adam）。「Adam 是練手用的；要打論文得試 SGD + Momentum」是業界笑話。
⚠ 易混概念釐清 · Adam vs SGD+Momentum（考古題 17 對應） 考古題常考「哪個優化器內建動量？」這題很容易答錯。重點釐清：

Momentum（動量）≠ SGD+Momentum（動量優化器）≠ Adam 的「第一階矩」

① SGD（純粹）：每步只用當下梯度，w = w - lr·∇L — 無動量
② SGD + Momentum：累積過去梯度當「慣性」，v = β·v + ∇L; w = w - lr·v — 這才是「動量優化器」的標準稱呼（β 通常 0.9）
③ Adam：= Momentum（第一階矩 m）+ RMSprop（第二階矩 v）+ 偏差校正 — 公式 w = w - lr·m̂/(√v̂ + ε)

關鍵：Adam 內含動量機制（第一階矩 m 等同 momentum 的累積），但它不是「動量優化器」 — 是「自適應學習率 + 動量」的融合體。考題若問「哪個是動量優化器」，答 SGD+Momentum；若問「Adam 是否含動量機制」，答 是。

新世代 ① · Lion · EvoLved Sign Momentum（Google, 2023）

SYMBOLIC PROGRAM SEARCH

新世代 ② · Sophia · Second-order Clipped（Stanford, 2023）

2× FASTER THAN ADAMW

原理：用輕量級的對角 Hessian 估計作為預條件子（preconditioner），再對更新量做逐元素 clipping。第一個能在 LLM pretraining 規模上真正打敗 AdamW 的二階優化器。

新世代 ③ · Muon · MomentUm Orthogonalized by Newton-Schulz（K. Jordan, 2024）

CURRENT WORLD RECORD

原理：先做 Nesterov momentum，再用 Newton-Schulz 迭代把 2D 參數更新「正交化」 — 比 SVD 更能在 GPU Tensor Cores 上高效執行。只用於 2D 隱藏層權重（embedding/輸出層仍用 AdamW）。

效能：2024 年提出；對 LLM 訓練可降 約 50% 成本、節省 33% 記憶體；達 AdamW 約 2× 計算效率；目前持有 NanoGPT 與 CIFAR-10 訓練速度世界紀錄。

意義：被許多研究者視為「Adam 以來第一個真正的根本性突破」 — Kimi K2（1.04T 參數 MoE）用 MuonClip 變體（Muon + QK-Clip）達成「零 loss spike」訓練 15.5T tokens。

📌 補充 · AdamW 為何成為 LLM 預訓練的事實標準 Adam 原版的「weight decay」其實是 L2 正則化的偽裝，在自適應學習率下會被梯度大小縮放，失去正則效果。AdamW（Loshchilov & Hutter, 2017, ICLR 2019）把 weight decay 從梯度更新中完全解耦，恢復原始 weight decay 語意。GPT-3、GPT-4、Llama 全系列、PaLM、BERT 全部採 AdamW — Llama 3.1 405B 配置 β₁=0.9、β₂=0.95、weight decay=0.1 訓練完 15T tokens。
📌 補充 · 記憶體優化雙雄 — 8-bit Adam & Adafactor Adam 每個參數要存 momentum + variance 共 8 bytes，使優化器狀態甚至比模型本身還大。8-bit Adam（bitsandbytes）用 block-wise 動態量化把 states 壓到 8-bit、記憶體降 75%；Adafactor則用 row × column 外積近似 variance，達成接近 16-bit Adam 的記憶體。兩者是 QLoRA 等單卡微調大模型工作流的關鍵基礎建設。
📌 補充 · Distributed Shampoo（Google → Meta, 2024）— AlgoPerf 冠軍 Shampoo 屬於 AdaGrad 家族，用 Kronecker 因子化的 block-diagonal preconditioner 近似 full-matrix AdaGrad。Meta 2024 發布的 PyTorch Distributed Shampoo 支援 DDP/FSDP/HSDP，在 AlgoPerf 基準競賽以最短 wall-clock 時間奪冠；4-bit 量化版本更可壓縮預條件子狀態約 7×。是少數能與 AdamW 競爭的「結構化二階方法」。

D · 收斂判準與訓練穩定性

你開始往山下走 — 但什麼時候該停？跟怎麼走才不會跌倒？這就是收斂判準與訓練穩定性的核心。山谷比喻中三種常見的「跌倒」場景：① 步伐太大（學習率過高）→ 跳過最低點還飛到對面山坡，loss 不降反升；② 步伐太小（學習率過低）→ 永遠走不到底，訓練時間爆炸；③ 該停沒停（過度訓練）→ 開始記住資料的雜訊，泛化到新資料反而變糟（過擬合）。

四種收斂判準

CRITERION 01
損失函數變化趨緩
連續多次迭代變化幅度極小（低於 ε） → 學習進入平緩，視為收斂
CRITERION 02
驗證集效能不再提升
驗證集指標持平甚至下降 → 達到泛化能力上限；再訓練可能過擬合
CRITERION 03
梯度趨近零
參數梯度持續逼近零 → 損失函數位於平坦區域
CRITERION 04
步數或時間達上限
運算資源有限或週期受控時，固定 epoch 或最大訓練時間；應搭配其他指標

原因後果

四種穩定訓練策略

① 學習率調整 Learning Rate Scheduling

LR SCHEDULE

隨訓練進行自動調降學習率，避免後期更新過大導致震盪、提升精細收斂能力。

常見策略：Step Decay（階梯式）、Cosine Annealing（餘弦衰減）、ReduceLROnPlateau（停滯時減半）。

② Early Stopping 提早停止

當驗證集效能在一段時間（連續 5~10 個 epoch）未改善 → 中止訓練，防止過擬合。

③ Gradient Clipping 梯度裁剪

為防梯度值過大（特別在 RNN 或深層模型），限制梯度的最大值，避免發散或數值不穩。

④ Batch Normalization 批次正規化

在每一層中標準化中間輸出，讓輸入分佈穩定，提升收斂速度與穩定性。

📌 補充 · Cosine Warmup — LLM 預訓練的事實標準 Loshchilov & Hutter 2016 提出的排程：先用幾千步線性 warmup 把學習率從 0 拉到 peak，再以餘弦曲線平滑衰減到 peak 的 10%。Llama 3.1 405B 採 8,000 步 warmup 接 1,200,000 步 cosine，從 8×10⁻⁵ 衰減到 8×10⁻⁷；GPT-3、Chinchilla、BLOOM、Falcon、Pythia也都採同一範式。Leslie Smith 2017 提出的 One Cycle Policy 則展示「高學習率本身就是正則化」— 常用於 CNN 視覺模型微調。
📌 補充 · BF16 混合精度 — 為什麼 LLM 預訓練幾乎都選 BF16 而非 FP16 BF16 與 FP32 共享指數位元寬度（8 bits），動態範圍與 FP32 相同，因此可「drop-in 替換」FP32 而不需 loss scaling、不需重新調超參數；FP16 雖然有 10 bits 尾數精度更高，但 5 bits 指數讓它在大模型訓練極易 overflow。Google TPU、NVIDIA A100/H100 都原生支援。這就是 GPT-4、Llama 3、Gemini、Claude 系列預訓練幾乎都選 BF16 而非 FP16 的根本原因。
📌 補充 · ZeRO / FSDP — 分散式優化的記憶體革命 傳統 Data Parallel 讓每張 GPU 都保留整份模型 + 梯度 + 優化器狀態，造成巨大冗餘。Microsoft DeepSpeed 提出的 ZeRO 把這三類狀態分階段切分（ZeRO-1 切 optimizer、ZeRO-2 再切 gradients、ZeRO-3 連 parameters 都切）。PyTorch FSDP（Fully Sharded Data Parallel）即是 ZeRO-3 的原生實作，是 2026 年訓練 100B+ 模型的兩大主流框架。沒有它，今天的 LLM 訓練不可能。
📌 補充 · Gradient Checkpointing — 以時間換記憶體 標準反向傳播必須保存所有前向 activations，在長序列大模型上 activations 可佔 90%+ 記憶體。Gradient Checkpointing 只在每個 Transformer block 邊界保留 activation，反向傳播時即時重算其餘部分，可省 60–80% 記憶體、僅多花約 20–30% 時間，是 QLoRA 與長上下文微調幾乎必開的選項。
📌 補充 · μP（Maximal Update Parametrization, Microsoft 2022）— 超參數零成本搬到大模型 透過特殊的 init / lr scaling 規則，讓網路寬度任意變化時「最佳超參數維持不變」。催生 μTransfer 範式 — 在便宜的小模型上完整 tune 超參數，再零成本搬到大模型，把 GPT-3 等級模型的調參成本砍掉九成以上，已成為當代基礎模型訓練的標準工具（arXiv 2203.03466, Tensor Programs V）。

📌 補充 · 蒙地卡羅方法在風險預測的應用（考古題 23 對應） 蒙地卡羅（Monte Carlo, MC）= 用「大量隨機抽樣」估算機率分佈。當「解析解算不出來」或「機率公式太複雜」時，MC 是萬能解法。

真實案例 · 太陽能發電風險預測：明天發電量 = 日照時數 × 板效率 × 雲量影響 × 溫度衰減 — 4 個變數都有不確定性，要算「明天發電量 < 5kWh 的機率」很難用公式解。MC 解法：
① 抽樣：對每個變數從歷史分佈隨機抽 10000 次
② 計算：每次抽樣算出該情境的發電量
③ 統計：10000 次中有多少次 < 5kWh → 那個比例就是風險機率
④ 信賴區間：取 5% 和 95% 分位數 → 給出「明天發電量 95% 信賴區間 = [4.2, 11.8] kWh」

其他 MC 應用：① 金融風險（VaR 計算）② 物理模擬（粒子運動）③ 強化學習（AlphaGo 蒙地卡羅樹搜尋 MCTS）④ 貝氏推論（MCMC 抽樣）⑤ 期權定價（Black-Scholes 數值解）。

iPAS 考點：看到「透過隨機抽樣估算機率分佈 / 風險區間」的題目 → 答案是蒙地卡羅方法，不是直接解公式。

從基礎原理開始 — 監督式、非監督式、強化式三大學習範式；接著進入經典演算法（迴歸、決策樹、SVM、集成學習）；最後深入深度學習架構（CNN、RNN、Transformer）。

機器學習原理與技術

機器學習（Machine Learning, ML）不依賴人為定義的規則，而是依據資料中的統計規律萃取模式與關聯，指導未來判斷。本節釐清 ML 的三大學習範式 — 監督式、非監督式、強化式 — 及其核心構成要素。

本節主軸 · 從「人類寫規則」到「機器從資料找規則」 1970s「專家系統」想把人類醫生 / 律師的知識寫成 IF-THEN 規則 → 慘敗。原因：真實世界有太多例外，規則永遠寫不完。ML 的革命性想法：別讓人類寫規則，讓機器自己從資料中找規則。本節要回答兩個問題：① 機器要學什麼形態的「規則」？（監督 / 非監督 / 強化三範式）② 一個 ML 系統由哪些零件組成？（五大核心構成）— 之後 §4.2、§4.3 會深入「怎麼學」的具體演算法。
§4.1 全節結構 · 三大學習範式怎麼分
三大範式的根本分野：「資料長什麼樣」決定你能用哪種學習 — 不是「想用什麼演算法」就用什麼

A · 機器學習的五大核心構成

要訓練一個 ML 模型，你不能只說「拿資料丟進去就好」 — 你必須把五件事想清楚：① 輸入是什麼？② 想預測什麼？③ 用哪個模型？④ 怎麼定義「對」與「錯」？⑤ 訓練好怎麼驗證？少任何一環，整個系統就會在某個點崩潰。下表的五大核心構成，就是任何 ML 專案動手前必須先回答的五個問題 — 也是 AI 應用規劃師在跟工程師對話時的共同語言。

構成要素說明
① 輸入資料與特徵空間結構化表格、文字、影像、語音、感測器資料 → 轉換為向量或矩陣。每個樣本是特徵空間（Feature Space）中的一個點
② 任務目標與標籤型態分類（離散標籤）、迴歸（連續數值）、聚類（無標籤分群）、降維、序列決策
③ 模型與假設空間把輸入映射為預測的數學函數；不同模型對資料分佈有不同的隱含假設（線性模型假設線性邊界、決策樹假設條件規則、NN 學非線性抽象）
④ 學習目標與損失函數透過損失函數量化預測與實際結果的誤差；設計反映任務偏好與風險取捨
⑤ 資料分割與評估準則訓練集 / 驗證集 / 測試集；分類用 Accuracy / Precision / Recall / F1，迴歸用 MAE / MSE / R²

三種學習範式對照

① 監督式學習（Supervised Learning）

想像小朋友學認動物 — 媽媽指著貓說「這是貓」、指著狗說「這是狗」幾百次之後，孩子自己看到新動物就能說出名字。監督式學習就是這個過程的數學版本：給機器一大堆「輸入 + 答案」配對（如「圖片 + 類別」），機器學會 X → Y 的映射函數。2026 年絕大多數商業 ML 應用 — 信用評分、客戶流失預測、CT 影像診斷、垃圾郵件分類 — 都是監督式。但它有個致命限制：標籤要人類標，一張醫療影像標一次要醫生 5 分鐘，所以「資料」永遠是瓶頸。

構成說明
資料每筆訓練資料由輸入特徵 + 對應目標標籤組成
學習目標最小化模型輸出與實際標籤之間的誤差，同時具備良好的泛化能力
主要任務分類（離散類別：垃圾郵件、流失預測、圖像辨識）；迴歸（連續數值：房價、銷售量、評分）
訓練流程① 資料準備與標記 → ② 模型訓練 → ③ 驗證與調整 → ④ 測試與部署
關鍵成敗因素資料的標註品質與特徵設計的精準性
📌 補充 · 自監督學習（Self-supervised）— LLM 的革命性「破局」 監督式的瓶頸是「需要人類標籤」 — 但 BERT（2018）與 GPT 系列開創了一條新路：讓資料本身產生標籤。BERT 做法：把句子「我今天去[MASK]吃飯」遮住一個字，讓模型猜「餐廳/公司」 — 整個維基百科都能變成數十億筆訓練樣本，零標註成本。GPT 做法更簡單：「給定前 N 個字，預測第 N+1 個字」，每篇文章自動拆出上千個訓練樣本。這就是為何 LLM 能用整個網際網路訓練 — 自監督打破了「資料 = 瓶頸」的詛咒。視覺領域也跟進：MAE（Meta 2021）遮住影像 75% 讓模型重建，無標註也能學會視覺特徵。

② 非監督式學習（Unsupervised Learning）

情境：電商老闆給你 100 萬筆顧客行為資料 — 但沒人知道顧客天然分幾類（5 類？50 類？）。這時你需要的不是「預測」（你不知道要預測什麼），而是「探索」 — 讓機器告訴你「資料裡藏著什麼自然分群、什麼異常、什麼降維後的低維結構」。這就是非監督式學習的精神：沒有老師告訴你答案，只能從資料本身找模式。難處在於「沒有標籤就難以評估好壞」 — 一個分群結果到底好不好？沒有絕對答案。

構成說明
資料無標註資料；只有 X，沒有 Y
學習目標發現潛在結構、模式、分佈特性
主要任務聚類（相似性分群）、降維（高維 → 低維）、關聯規則學習（共現規律）、潛在表示學習（語言模型、影像編碼）
評估方式內部指標（Silhouette score 輪廓係數、Davies-Bouldin、Reconstruction Error）；外部指標（Rand Index、AMI）；視覺化輔助
應用領域客戶分群、社群偵測、影像壓縮、主題模型、特徵工程前處理

③ 強化式學習（Reinforcement Learning）

想像你教一隻狗坐下：你不會給它一本「坐下使用手冊」（規則），也不會給它「100 張正確坐姿照片 + 100 張錯誤坐姿照片」（監督式）— 你只會在它坐對時給零食、坐錯時什麼都不給。一段時間後，狗自己摸索出「坐下會得零食」的策略。這就是強化式學習：代理（狗）在環境（你）中嘗試動作（坐/站），透過回饋（零食 = 獎勵）逐步學到最佳策略。AlphaGo、Tesla FSD、ChatGPT 的 RLHF、YouTube 推薦 — 都是這套機制的工業實踐。

真實案例 · ChatGPT 是怎麼變「會聽話」的？RLHF 三部曲

2022 BREAKTHROUGH

背景：GPT-3（2020）能力強大但「答非所問」 — 給它「解釋光合作用」常常答出「我餓了」。怎麼把強的模型變得聽話？答案：強化學習。

RLHF（Reinforcement Learning from Human Feedback）三階段：

  • 階段 ① · 預訓練（Pre-training）：用整個網際網路訓練 GPT-3 — 自監督，學會語言基本能力。強但不聽話。
  • 階段 ② · 監督式微調（SFT）：40 人團隊寫數萬筆「示範回答」，讓模型模仿「人類會怎麼回應」。開始有禮貌。
  • 階段 ③ · 強化學習對齊（RLHF）：同一個 prompt 讓模型生成 4-7 個答案，人類標註員排序好壞。用這些排序訓練「獎勵模型」當代理的環境，再用 RL（PPO）讓 GPT 朝獎勵高的方向學。變成 ChatGPT。

後續演進：Anthropic 的 Claude 改用 RLAIF（讓 AI 取代人類標註，按「憲法 Constitution」評分），成本降數十倍。2023 Stanford 提出的 DPO（Direct Preference Optimization）更激進 — 直接用偏好資料優化，省掉獎勵模型階段，比 PPO 快 2-10 倍且更穩，成為當前主流。

教學意義：從 GPT-3 到 ChatGPT 不是「模型變大」，而是「用強化學習把人類偏好灌進去」 — 這就是為什麼 §4.1 強化學習不能跳過。

反直覺案例 · 哈士奇 vs 狼 — 過擬合最經典的寓言

RIBEIRO 2016 · LIME PAPER

故事：2016 年華盛頓大學研究團隊訓練了一個「哈士奇 vs 狼」分類器，準確率 90% 看似很好。但用 LIME 工具拆解模型決策後，結果震撼：

模型根本不看動物，它只看「背景有沒有雪」 — 訓練集裡狼的照片都在雪地、哈士奇都在室內，模型聰明地學會了這個「偽相關（spurious correlation）」偷吃步。把哈士奇放在雪地裡 → 立刻被判為狼。

為何這故事這麼重要：

  • 準確率高 ≠ 模型真的學會了你以為它學到的東西
  • 模型會找「最容易讓 loss 降低」的捷徑 — 即使這個捷徑跟任務無關
  • 這篇論文（KDD 2016）引用數超過 18,000 次，是可解釋 AI（XAI）的起源故事
  • 商業案例：你以為訓練了一個「客戶流失預測模型」，但它其實只學到「客戶帳號是 2018 年註冊」這個無關特徵 — 上線就崩
📌 補充 · 醫療 AI 的 Domain Shift — 99% 變 60% 的崩潰 2018 年知名研究發現：用某醫院 X 光資料訓練的肺炎 AI，自家醫院測試 AUC 0.93，搬到另一家醫院掉到 0.82，再換一家更慘。深入追查才發現：模型偷偷學會了辨認「這是哪家醫院的 X 光機」 — 因為各家醫院的肺炎盛行率不同，模型用「機器特徵」就能猜中。這就是 domain shift：實驗室 99% 準確，到了另一家醫院可能只有 60%。這是台灣醫療 AI 落地最大的隱形殺手 — AI 應用規劃師必須事先在「多家醫院的資料」上做外部驗證，不能只看訓練集表現。
📌 補充 · 時間序列資料洩漏 — 為何不能隨機 split 想像你要訓練股價預測模型，把 2020–2024 五年資料隨機 80/20 切 train/test — 大錯。隨機切會讓「2024 年的資料」混進訓練集、「2020 年的資料」混進測試集，等於用未來預測過去。研究實測：LSTM 預測任務中，隨機 10-fold CV 比正確的時序切分高估準確度 20.5%（RMSE gain）— 回測 95% 準確、上線就崩。正確做法是 walk-forward：只用過去預測未來。
§4.1 小結 · 一頁讀懂 ML 三大範式 + 五大構成 ① 從規則式 AI 到統計 ML：1990s 後典範轉移 — 別讓人類寫規則，讓機器自己學；② 三大範式 = 三種資料形態：監督式（有標籤）/ 非監督式（只有 X）/ 強化式（互動 + 回饋）；③ 五大核心構成：輸入特徵、任務目標、模型假設、損失函數、評估方式 — 缺一不可；④ 2024-2026 新範式延伸：自監督（BERT/GPT/MAE — 從資料本身產生標籤）、Few-shot/Zero-shot（LLM 不訓練就學）、RLHF（讓 ChatGPT 聽話）、AlphaGo→AlphaProof 系列（RL 攻克科學難題）；⑤ 實務陷阱：哈士奇-狼故事（偽相關）、醫療 domain shift（99%→60%）、時序資料洩漏 — 都源自「監督式學習以為高準確率就是真的學會」。

A · 迴歸類演算法

迴歸的目標是預測一個連續數值 — 房子值多少錢、明天會下幾毫米雨、客戶生命週期價值會是多少。最常見的場景是「我有歷史資料」（如過去一年每天的銷售數字 + 當天的廣告支出、天氣、節慶），想預測未來。本組演算法從最簡單的線性迴歸（一條直線）到當代 Kaggle 競賽王者 XGBoost（百棵樹的集成），技術跨度極大，但核心目標都是「給定輸入特徵，輸出一個準確的連續數字」。

① 線性迴歸 Linear Regression

核心：用直線或多維超平面描述自變數與應變數的線性關係。

y = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ + ε

五大假設：① 線性關係 ② 誤差常態分佈 ③ 變異數齊一性（Homoscedasticity）④ 誤差獨立 ⑤ 無多重共線性。

評估：MSE / RMSE / MAE / R² / 調整後 R²。

適用：連續數值預測；變數關係接近線性；需要可解釋性（金融、醫療）。

限制：對離群值極敏感；無法捕捉非線性關係；多重共線性使係數估計不穩。

② Lasso 迴歸 與 嶺迴歸 Ridge

L1 / L2 REGULARIZATION

在損失函數加入正則化項抑制過擬合：

Lasso（L1）：Σ(y − ŷ)² + λ Σ|βⱼ|　　Ridge（L2）：Σ(y − ŷ)² + λ Σβⱼ²

關鍵差別：

  • Lasso（L1）：能將部分係數縮減為零 → 達到特徵選擇效果
  • Ridge（L2）：限制係數大小但不為零；特別適合多重共線性嚴重的情況

適用：變數數量大於樣本數（高維資料）、多重共線性、需特徵篩選。

限制：Lasso 對高相關變數可能只保留其一；Ridge 無法自動特徵選擇；λ 需透過交叉驗證選擇。

③ 支援向量迴歸 SVR

Support Vector Regression

核心：在多維空間中尋找最能描述資料趨勢的超平面，使預測誤差落在 ε-tube 內，超出範圍才懲罰。

f(x) = w · x + b　（線性）　　f(x) = Σ αᵢ K(xᵢ, x) + b　（非線性）

關鍵參數：C（懲罰參數）、ε（誤差容忍範圍）、γ（核函數參數）。

優勢：對離群值有抗性（多數小誤差被忽略）；可用核函數處理非線性。

限制：計算複雜度高（樣本平方增長）；不易解釋；不適合極大規模資料。

④ 決策樹迴歸 Decision Tree Regressor

核心：依特徵分割資料成更小的子區塊，葉節點預測值為該區塊內樣本的平均值。

優勢：不需常態假設、不需標準化；可處理類別+數值混合特徵；捕捉非線性與高階交互作用；解釋性強。

限制：容易過擬合；預測為分段常數（不連續）；對資料微小變動敏感（結構易變）。

⑤ 集成式迴歸 — 隨機森林 / 梯度提升 / XGBoost / LightGBM

ENSEMBLE

群體智慧比單一模型更強大 — 多個模型不同角度學習，整合結果降低錯誤、減少過擬合、提升泛化能力。

方法原理
隨機森林
Random Forest		平行式建多棵決策樹（不同 Bootstrap 樣本 + 隨機特徵子集）；最終取平均。抗過擬合強、可並行加速、計算特徵重要性
梯度提升
Gradient Boosting		序列式逐步修正誤差 — 訓練一棵樹 → 算殘差 → 下一棵樹學殘差。預測精度通常高於隨機森林，但訓練慢
XGBoost強調運算效率與正則化；用二階導數（Hessian）加速分裂點搜尋；層級式生長（Level-wise）；支援稀疏資料與缺失值
LightGBM葉子式生長（Leaf-wise）策略 — 優先分裂最大增益的葉節點；用直方圖算法降低計算複雜度；對大資料/大特徵尤其高效
補充：XGBoost vs LightGBM 的選擇邏輯 XGBoost：層級式生長 → 樹更均衡，較不易過擬合，適合中型資料。
LightGBM：葉子式生長 → 樹更深更不均衡，速度更快、精度通常更高，但需 max_depth 限制以防過擬合，特別適合大資料、大特徵數。
業界共識：先試 LightGBM（更快），若效果不穩或樣本少改用 XGBoost。CatBoost 是第三選擇 — 對類別特徵有特殊處理。

真實案例 · XGBoost — 結構化資料的「十年不敗王者」

CHEN 2014 · KAGGLE DOMINANCE

背景：陳天奇 2014 年在華盛頓大學發表 XGBoost（eXtreme Gradient Boosting），是 Gradient Boosting Decision Tree（GBDT）的高效實作。發表後迅速統治 Kaggle 競賽 — 至今仍是結構化資料的 SOTA。

數據：

  • 2015-2025 十年間，超過 70% Kaggle 結構化資料競賽冠軍使用 XGBoost
  • 2024 Kaggle 統計：XGBoost 8 場冠軍、LightGBM 16 場、CatBoost 13 場 — GBDT 三巨頭共佔 85% 冠軍
  • 2025/04 Playground 冠軍 Chris Deotte 用 72 模型三層 stacking，核心仍是 GBDT 三巨頭
  • 能在筆電 CPU 上 5 分鐘內訓練完 100 萬筆資料，且準確率往往打敗動輒需要 GPU 數小時訓練的深度學習模型

三巨頭分工：

  • XGBoost：穩定可靠、調參彈性大、社群最大 — 像「瑞士刀」
  • LightGBM（Microsoft）：histogram-based + leaf-wise，速度可達 XGBoost 7 倍 — 像「跑車」
  • CatBoost（Yandex）：類別特徵免做 one-hot encoding 直接吃 — 電商廣告場景首選

啟示給 AI 應用規劃師：「在 LLM 時代仍要懂 XGBoost」— 因為企業內部 80% 的 ML 應用（信用、流失、需求預測）都是表格資料，LLM 在此完全打不過 XGBoost。第一準則：資料是表格 → 先試 XGBoost。

📌 補充 · TabPFN — 表格資料的「ChatGPT 時刻」?（2024-2026 新興） 2024 年 Prior Labs 釋出 TabPFN：基於 Transformer 架構的表格資料基礎模型，預訓練於合成資料，無需訓練即可推論 — 就像 ChatGPT 對文字一樣，TabPFN 直接「讀」一張表然後預測。TabPFN-3（2026/05）支援 100 萬列、200 特徵，在小於 10,000 樣本的資料集上「零訓練、秒級推論」打敗 XGBoost。雖然在大資料與深度調參上仍輸 GBDT 三巨頭，但它預示「表格資料的 ChatGPT 時刻」可能在 2027 到來 — 屆時資料科學家可能不需要再調參數，直接「prompt」一張表就得到模型。

B · 分類類演算法

分類的目標是把樣本歸到離散類別之一 — 這是封信是垃圾還是正常？這位客戶會不會流失？這張 X 光有沒有腫瘤？分類演算法是 ML 最廣泛應用的領域。看似簡單，但要做好需要回答多個關鍵問題：類別之間的邊界是線性的還是非線性的？（線性邊界 → 邏輯迴歸 / SVM；非線性 → 樹模型 / 深度學習）類別之間平衡嗎？（不平衡需要 Focal Loss、SMOTE）需要可解釋嗎？（金融醫療 → 決策樹 / 邏輯迴歸）下方七個演算法各有強項，沒有單一「最好」 — 只有「最適合該情境」。

分類演算法最直觀的差別在於「畫出來的決策邊界長什麼樣」 — 同一筆 2D 資料丟給不同演算法，畫出的邊界形狀完全不同。下圖是 ML 教材界最經典的視覺對比（取材自 sklearn 官方文件範例），一張圖看完六種演算法的「個性」：

分類演算法決策邊界對比 · 同一筆「雙月形」資料丟給六種演算法
同一筆「雙月形」資料 × 六種演算法 = 六種完全不同形狀的邊界 — 「演算法選型」本質就是「選一個邊界形狀」

① 邏輯迴歸 Logistic Regression

核心：雖名為「迴歸」實為分類；用 sigmoid 函數把線性組合映射到 [0, 1] 機率區間。

p = 1 / (1 + e−(β₀ + β₁x₁ + … + βₙxₙ))

假設：① 對數勝率（Log-Odds）與特徵線性；② 資料獨立；③ 無多重共線性。

評估：Accuracy、Precision、Recall、F1、ROC-AUC。

適用：二元分類（購買/不購買、患病/健康）；需要可解釋；金融/醫療場景。

限制：無法捕捉複雜非線性邊界；對離群值敏感；多類別需擴展（One-vs-Rest 或 Multinomial）。

② 支援向量機 SVM

核心：在特徵空間中尋找能區分不同類別的最佳超平面，最大化兩類之間的間隔（Margin）。

w · x + b = 0　　決策依 w · x + b 的正負

四大常用核函數：線性核、多項式核、RBF（Gaussian）核、Sigmoid 核。

關鍵：核函數讓 SVM 可把原本不可線性分隔的資料映射到更高維度使其可分。

適用：資料規模中小、特徵維度高、邊界清晰、需精準分界。

限制：計算成本高（大資料訓練慢）；參數調整複雜（C、γ）；多類別需擴展。

③ 決策樹分類器 Decision Tree

核心：透過「如果…那麼…」判斷規則分割資料，葉節點給出類別預測。

分裂依據（衡量純度）：

  • 基尼不純度（Gini Impurity）：節點內混雜程度；越小越集中於單一類別
  • 資訊增益（Information Gain）：依熵（Entropy）變化衡量分裂後不確定性降低多少
  • 分類誤差（Classification Error）：節點內分錯的比例

優勢：非線性、無需標準化、處理混合型特徵、解釋性強。限制：容易過擬合；對資料變動敏感。

④ 集成式分類 — Random Forest / Boosting

與迴歸版本原理相同 — 隨機森林取多數投票、梯度提升序列修正錯誤。XGBoost / LightGBM 在分類任務上同樣是 Kaggle 競賽常勝軍。

⑤ K 最近鄰 KNN

K-Nearest Neighbors

核心：新樣本的類別 = 訓練集中最接近的 k 個鄰居的多數類別。

關鍵：不需訓練（懶惰學習）；對距離度量（歐幾里得、曼哈頓、餘弦）敏感；對 k 值與特徵尺度敏感。

適用：小型資料集、特徵空間分佈明顯的情境。限制：記憶體需求高（需存全部訓練資料）；對量與維度敏感（維度詛咒）。

⚠ 演算法複雜度 · 為何 KNN 在大資料下不可用（考古題 33 對應） KNN 沒有訓練階段，但預測時要跟全部 n 筆訓練資料算距離 — 預測 m 個新樣本要 O(m·n·d) 時間（d 是維度）。當 n = 100 萬時，每筆預測要算 100 萬次距離，嚴重不可擴展。

⑥ 樸素貝氏 Naive Bayes

核心：基於貝氏定理 + 特徵獨立性的「樸素」假設。

P(類別 | 特徵) ∝ P(特徵 | 類別) · P(類別)

變體：Gaussian NB（連續特徵）、Multinomial NB（文字計數）、Bernoulli NB（二元特徵）。

適用：文字分類（垃圾郵件、情感分析）；訓練快、所需資料少。限制：「特徵獨立」假設在現實中常不成立。

C · 聚類類演算法（非監督式）

想像電商老闆給你 100 萬筆顧客資料：年齡、購買頻率、購物車金額、瀏覽時長…他問：「我的顧客有幾種類型？」沒有人事先告訴你答案 — 因為沒人知道。聚類演算法就是這個情境的解 — 讓資料自己告訴你「自然分成幾群」。最有名的聚類演算法是 K-means，但它有三大致命弱點，導致在許多真實場景失靈 — 這正是 DBSCAN、HDBSCAN、Gaussian Mixture 等演算法存在的理由。

D · 降維類演算法（非監督式）

想像你要在 Google Maps 上標記「2,000 篇文章」的彼此相似性 — 但 OpenAI text-embedding-3 把每篇文章變成 1,536 維向量。1,536 維在 2D 螢幕上根本不可能畫出來。降維演算法的工作就是 — 把高維資料壓到 2D 或 3D，同時盡量保留「誰跟誰相似」的結構。這在 LLM 時代特別重要：你想看 ChatGPT 的 embedding 如何把「橘子」、「蘋果」、「水果」聚在一起，把「香蕉」、「黃色」、「猴子」連結，UMAP 就是 2025 業界標準工具，速度比 t-SNE 快 5-10 倍且能保留全域結構。

反直覺案例 · Isolation Forest — 用「樹深」找異常的反向思維

LIU 2008 · ANOMALY DETECTION

反直覺起點：異常偵測的傳統思路是「先學什麼是正常，偏離正常就是異常」（如 One-Class SVM 圈出正常區）。但劉飛 2008 年提出 Isolation Forest，反其道而行：「隨機砍樹，看哪些點最快被切到單獨葉子」。

直覺：異常點本來就少且離群，砍幾刀就被孤立；正常點擠在一起，要砍很多刀才能切開。最終以「樹的平均深度」判斷異常分數 — 深度越淺 = 越異常。

為何反直覺卻有效：

  • 不需要距離度量 — 在高維資料（如 100+ 特徵的詐欺交易）上比距離型方法（DBSCAN）效率高百倍
  • 線性時間複雜度 O(n)，能毫秒級處理百萬筆交易
  • 對不平衡資料天然有利（異常本來就少）

業界應用：Stripe / PayPal 的即時詐欺偵測核心元件、IoT 感測器異常監測、網路入侵偵測標配；scikit-learn 內建。

教學意義：不是所有 ML 問題都該「擬合正常分布」 — 有時候「找最特立獨行的點」反而更直接。這是 ML 思維中「反向設計」的經典案例。

📌 補充 · 集成學習的「群眾智慧」反直覺 集成學習有個數學奇蹟：若每個基學習器準確率只有 51%（略勝隨機），10 個獨立投票後準確率約 75%，100 個則約 97%。這正是 Condorcet 1785 年「陪審團定理」的核心，也是「群眾智慧」現象。但有個關鍵前提：「弱學習器要彼此獨立」。Random Forest 用 bagging（隨機抽樣 + 隨機特徵）強制獨立性，XGBoost 用 boosting 序列補強，Stacking 則用「元學習器」整合不同型態模型。2025 Kaggle 冠軍解法常見「70 個模型三層 stacking」 — 正是此原理的極致實踐。這也是為什麼 §4.3 Mixture of Experts (MoE) 和 Multi-Agent LLM 仍是同一個邏輯的演進。
📌 補充 · 邏輯迴歸 — 不死的「基準線」（金融與醫療場景） 2026 年了，為什麼銀行算信用分數還在用邏輯迴歸？因為《巴塞爾協定》與台灣金管會明文要求「決策邏輯需可解釋」。LR 的每個係數都直接對應「特徵 X 對風險的貢獻」，且訓練只需數秒、能在邊緣裝置上即時推論。研究顯示 RF 雖在 69% 資料集上贏過 LR，但剩下 31% 場景 LR 不輸 — 加上「可解釋」這條鐵律，使 LR 成為任何 ML 專案的「必跑基準線」。AI 應用規劃師鐵則：先用 LR 跑基準，再決定是否升級到複雜模型。
📌 補充 · AutoML 2025 — 自動化選演算法的成熟 H2O AutoML、AutoGluon、FLAML、Auto-sklearn 等框架自動完成「資料前處理 → 演算法選擇 → 超參數調優 → 集成」全流程。2023 OpenML AutoML Benchmark：AutoGluon 1.0 平均排名 1.95（63% 拿第一）。AWS、Google Cloud、Azure 都內建 AutoML 服務 — 用戶上傳 CSV、按一鍵、30 分鐘後得到一個準確率 90% 的模型。意味著未來「跑模型」的技術門檻會持續降低，AI 應用規劃師的價值將從「會調 XGBoost 參數」轉移到「定義對的問題、評估資料品質、規劃落地流程」。
§4.2 小結 · 14+ 演算法的選用指南 ① 結構化資料的王者：XGBoost / LightGBM / CatBoost — 連續 10 年統治 Kaggle 70%+ 冠軍，企業 80% ML 應用首選；② K-means 三大死穴：非凸群、不同密度、含離群值 → 改用 DBSCAN / HDBSCAN / GMM；③ 降維三劍客：PCA（線性快速）/ t-SNE（視覺化精緻）/ UMAP（2025 標準，快 5-10×）；④ 異常偵測反向思維：Isolation Forest 用「易孤立」找異常，比「擬合正常」更有效；⑤ 群眾智慧數學奇蹟：弱學習器 51% × 100 個 → 97%；⑥ 邏輯迴歸不死老兵：受監管產業必跑基準；⑦ 2026 新趨勢：TabPFN（表格資料的 ChatGPT 時刻）+ AutoML（自動化選演算法）+ Tabular Foundation Models（DL for tabular）。

深度學習原理與框架

深度學習（Deep Learning）相較傳統 ML 不需手工特徵工程 — 透過多層神經網路自動學習數據中的高階抽象特徵，大幅提升影像、語音、NLP、生成式 AI 的表現。本節介紹基礎概念 → 五大代表架構（MLP、CNN、RNN、Transformer、生成模型）→ 主流框架。

本節主軸 · 為什麼深度學習花了 60 年才爆發？ 1958 年 Rosenblatt 發明感知器、《紐約時報》宣稱「電子大腦」即將誕生 — 結果 1969 年被 Minsky 用 4 個點的 XOR 問題擊倒，AI 寒冬整整 17 年。1986 年反向傳播救活神經網路，但又等了 26 年到 2012 AlexNet 才真正爆發。「為何花了 60 年」這個問題的答案是：缺三件事 — ① 演算法（反向傳播 1986、Adam 2014、Transformer 2017）；② 資料（ImageNet 2009）；③ 算力（GPU CUDA 2007、Tensor Cores 2017）。本節要帶你穿越這 60 年歷史，理解每個關鍵突破為何重要 — 2024 年諾貝爾物理獎 + 化學獎雙料頒給 AI 研究者（Hinton、Hassabis），正式為這段歷史加冕。

A · 神經網路基礎

神經網路的概念很簡單：模仿大腦的神經元，把多個「人工神經元」連成網路，輸入訊號流過，最後輸出預測。但這個簡單的概念背後有兩段史詩級的低谷與復興。Frank Rosenblatt 1958 年在康乃爾航空實驗室發表感知器（Perceptron），《紐約時報》當時宣稱：「電子大腦能行走、說話、看見、書寫、自我複製」 — 海軍出資、媒體狂熱、學界期待人工智慧即將降臨。然後在 1969 年，被 Minsky 和 Papert 寫的一本書、用一個只有 4 個點的 XOR 問題擊倒，整個領域沉寂 17 年。理解這段歷史，你才能理解為何後來「多層神經網路」與「反向傳播」是如此革命性。

① 人工神經元與感知器

人工神經元 Artificial Neuron

受生物神經元啟發的數學模型。接收多個輸入訊號 → 加權求和 → 非線性激活 → 輸出。

Z = w₁x₁ + w₂x₂ + … + wₙxₙ + b　　Output = A(Z)
  • 輸入 x：來自其他神經元或外部資料
  • 權重 w：每個輸入的重要性
  • 偏置 b（Bias）：允許神經元在沒有輸入時也能被激活
  • 激活函數 A：引入非線性 — ReLU、Sigmoid、Tanh

感知器 Perceptron

最早、最簡單的人工神經元模型（Frank Rosenblatt 1958 提出）。是二元分類器，使用步階函數作為激活：

f(x) = 1 if x ≥ 0；f(x) = 0 if x < 0

限制：只能解決線性可分問題，無法解決 XOR 問題 — 這個限制催生了多層神經網路的研究。

XOR 殺死神經網路 17 年的「4 個點」反直覺反例
紅 = Label 1 / 藍 = Label 0 · AND/OR 可線性分（紫虛線），但 XOR 怎麼畫直線都分不開 — 整個 AI 領域因此被擱置 17 年

② 激活函數的核心角色

激活函數特性
ReLU輸入為正直接輸出，為負輸出 0；稀疏激活，計算快，深度學習主流
Sigmoid壓縮至 [0, 1]；可解釋為機率；早期常用，但深層網路易梯度消失
Tanh壓縮至 [-1, 1]；零中心化，RNN 中常用
Softmax把向量轉為機率分佈（總和為 1）；多類別分類的輸出層
Leaky ReLU / GELU / SwishReLU 的改良；解決負區梯度為零的「Dying ReLU」問題
為什麼激活函數必須非線性？ 若沒有激活函數，無論神經網路有多少層，最終輸出都只是輸入的線性組合 — 等同單層線性模型，只能解線性可分問題。引入非線性激活函數後，神經網路才能逼近任意複雜的非線性函數。

B · 前向傳播 vs 反向傳播 — 神經網路如何學習

前向傳播（做預測）vs 反向傳播（修正錯誤）
圖 4-DL-1 · 前向 = 做預測；反向 = 用鏈式法則計算梯度並更新權重
階段說明
前向傳播
Forward		① 輸入資料進入第一層 → ② 每層加權求和 + 激活 → ③ 一層一層向前推進 → ④ 輸出層產生預測
計算損失把預測與真實標籤比較，用損失函數計算量化的誤差值
反向傳播
Backpropagation		用連鎖律（Chain Rule）把誤差從輸出層反向傳遞，計算損失函數對每個權重的梯度
權重更新優化器（SGD/Adam）根據梯度與學習率，沿梯度相反方向微調參數
迭代學習前向 + 反向重複數千至數百萬次，模型越來越精準

② 五大正則化技術

技術原理
L1 正則化（Lasso）損失函數加入「權重絕對值總和」；促使部分權重為零，達特徵選擇

Dropout訓練時隨機屏蔽部分神經元，避免神經元彼此過度依賴；測試時恢復全部
Early Stopping驗證集損失不再下降時提前停止訓練

D · 五大深度模型架構

理解深度學習的關鍵，不是記住所有架構名稱，而是理解每個架構為何被發明 — 它解決了前一代的什麼痛點。每一代的革命都是一次「打破不可能」：MLP 解決「非線性問題不能學」、CNN 解決「影像太大太慢」、RNN 解決「序列順序丟失」、LSTM 解決「RNN 記憶太短」、Transformer 解決「RNN 不能並行」、MoE 解決「模型再大算力不夠」 — 一場接著一場的接力賽。下方五大架構是 DL 的基本款式，但 2024-2026 主流模型（Llama 4、DeepSeek-V3、Kimi K2）都已是它們的進階變體。

深度學習架構選型決策樹 · 從「資料/任務類型」到「該選哪個架構」
紫色 = 分類問題 / 黃色 = 子問題 / 綠色 = 2024-2026 主流首選 — 看資料類型決定大類，子問題收斂到具體架構

① 多層感知器 MLP — Multilayer Perceptron

結構：輸入層 + 多個隱藏層 + 輸出層；每層之間全連接（Fully Connected）；資訊單向流動。

優勢：非線性建模能力強；廣泛適用結構化資料。

限制：① 會把空間/序列資訊「攤平」（丟失像素相對位置或詞序）；② 參數量大、計算成本高；③ 易過擬合；④ 黑箱缺乏可解釋性。

適用：結構化資料分類與迴歸（客戶流失、信用評分、房價）；作為其他複雜模型的後置分類器。

② CNN 卷積神經網路 — Convolution Neural Network

核心組件：

  • 卷積層（Convolutional Layer）：用卷積核滑動提取局部特徵 → 特徵映射（Feature Map）；權重共享減少參數、實現位置不變性
  • 激活層：通常用 ReLU 增加非線性
  • 池化層（Pooling Layer）：最大池化或平均池化降維、增強泛化
  • 全連接層：將提取的特徵整合進行最終分類

優勢：權重共享 + 局部連接（利用空間相關性）；對平移、輕微縮放旋轉具不變性。

限制：計算資源大；需大量標註資料；對較大旋轉/尺度變化仍受限。

代表模型特點
AlexNet (2012)深度 CNN 奠基者；ReLU + Dropout + GPU 加速；ImageNet 突破
VGG極深架構；多個 3×3 小卷積核堆疊
GoogLeNet (Inception)Inception 模塊；同層平行不同大小卷積核
ResNet (2015)殘差連接（Skip Connection）；解決深層網路梯度消失；可達 152 層
DenseNet每層與所有前層連接；特徵重用、緩解梯度消失
MobileNet / EfficientNet輕量級；深度可分離卷積；適合行動裝置與邊緣部署
CNN 三大武器視覺化 · 卷積運算 + 權重共享 + 池化（為何能用 9 個權重打敗 25 個輸入）
紫 = 當前 3×3 窗口 · 紅 = 卷積核（9 個參數） · 綠 = 輸出 Feature Map · 黃 = 計算過程 — CNN 用 9 個權重「掃」過整張圖，這就是「權重共享」的力量

③ RNN 遞迴神經網路 — Recurrent Neural Network

核心：處理可變長度序列資料；神經元之間有循環連接，允許資訊在網路內部持續流動，賦予「記憶」能力。

結構：每時間步接收當前輸入 Xₜ + 前一時間步隱藏狀態 Hₜ₋₁，產生新隱藏狀態 Hₜ 與輸出 Oₜ。

關鍵特性：① 權重共享（不同時間步用同一套參數）；② 「展開」視角（在時間維度展開為前饋網路）；③ BPTT（時間上反向傳播）訓練。

致命缺陷：

  • 梯度消失：長序列下梯度指數衰減，難學遠距離依賴 — 記憶太短
  • 梯度爆炸：梯度指數增長 → 訓練不穩；需梯度裁剪
  • 難並行：序列性導致計算串行，訓練慢
RNN 變體解決方案
LSTM
Long Short-Term Memory		三個門控機制（輸入/遺忘/輸出門）+ 細胞狀態；選擇性允許資訊流入、保留、移除，解決梯度消失
GRU
Gated Recurrent Unit		LSTM 簡化版；只有兩個門（更新/重置）；結構緊湊、效能相當
Bi-RNN
雙向 RNN		同時從前向後 + 從後向前處理序列；捕捉雙向上下文
Deep RNN多層 RNN 堆疊；學更複雜的時序特徵

應用：機器翻譯、語音辨識、時間序列預測（股價、天氣）、文本生成、情感分析、NER、動作辨識。

RNN 的「記憶」與「失憶」 · 時間展開圖 + 梯度消失機制
紫 = RNN cell / 綠 = 輸入 x / 紅 = 輸出 y · 反向梯度愈往左愈淡 — 走過 7 步就剩 1.5% — RNN 「記不到 10 步前的資訊」是物理現實，不是工程缺陷

④ Transformer — Attention Is All You Need (Google, 2017)

革命性突破：完全捨棄 RNN 循環結構，純粹基於注意力機制（Attention Mechanism）。能並行處理序列、透過自注意力捕捉任意位置的長距離依賴。

架構：Encoder 堆疊 + Decoder 堆疊（每個堆疊由多層相同的編解碼器層組成）。

  • Encoder（編碼器）：分析原始輸入序列，每個元素轉為富含上下文資訊的高維表示
  • Decoder（解碼器）：基於 Encoder 的上下文表示與已生成的部分序列，逐步生成目標輸出

六大核心組件：

組件功能
注意力機制
Attention		賦予模型動態「關注」序列中其他元素的能力；計算當前元素與所有其他元素的相似度
自注意力
Self-Attention		同一序列內元素間的關係；用 Query/Key/Value 三個向量計算 — Q「我在找什麼」、K「我能提供什麼」、V「我的內容」
多頭注意力
Multi-Head Attention		並行執行多個獨立的自注意力（多個「頭」）；每頭學習不同類型的關係；最後拼接
位置編碼
Positional Encoding		因為沒有循環結構，模型本身不感知順序 → 用正弦/餘弦函數加到詞嵌入，提供位置資訊
前饋網路 FFN兩個線性層 + ReLU；對每個位置獨立應用，進一步轉換注意力輸出
殘差連接 + 層歸一化跳躍連接緩解梯度消失；層歸一化穩定訓練（不依賴批次大小，適合序列）

應用：NLP 革命性架構 — BERT（雙向理解）、GPT（自迴歸生成）、T5、BART；近年也擴展到 Vision Transformer（ViT）、多模態（CLIP、Flamingo）。

Self-Attention 在算什麼？ · 一張 Attention 矩陣看懂「她」指誰
紅 Q = 我在找誰 · 綠 K = 我能提供什麼 · 紫 V = 我的內容 · 矩陣愈深紫 = 關注度愈高 — 「她」的 row 在「小華」column 點亮 0.65，模型就「懂」了指代
Transformer 革命的真正魔法：「並行化」打敗 RNN
紅 = RNN 串行（前一步沒完不能算下一步）· 綠 = Transformer 並行（所有 token 同時看彼此）— 速度差距百倍以上
📌 補充 · MoE 革命（2024-2025）— DeepSeek、Llama 4、Kimi K2 為何放棄 dense？ Mixture of Experts（MoE）把 Transformer 的 FFN 層拆成數百個「專家子網路」，每個 token 只由路由器選 2-8 個專家計算 — 好比一家綜合醫院，每位病患只看相關科別。DeepSeek-V3：671B 總參數 / 37B 激活、256 個專家、訓練成本約 $560 萬美元，比肩 GPT-4，震驚 2025 年初市場；Llama 4 Maverick：400B 總 / 17B 激活；Kimi K2：1T 總 / 32B 激活、384 個專家。2025 年已超過 60% 的開源 LLM 採用 MoE — 它讓「萬億參數」變得可訓也可部署。
MoE 稀疏激活機制 · 用「綜合醫院 vs 專科分診」理解為何 671B 模型只要 37B 算力
紅 = Dense（每 token 跑 671B）· 紫 = MoE（每 token 只跑 37B，但有 671B 容量可選）— 同一個模型，計算量差 18×，這就是 2024-2025 開源 LLM 爆發的工程基石
📌 補充 · Mamba 與 State Space Models（2023-2025）— Transformer 真不可取代？ 2023 年底 Mamba（Albert Gu, Tri Dao）橫空出世，挑戰 Transformer 七年的霸權：採用「狀態空間模型」(SSM)，計算複雜度從 Transformer 的 O(N²) 降為 O(N) 線性 — 理論上能處理百萬 token 不爆顯存。Mamba-3B 在語言建模匹敵 2 倍大的 Transformer，在 DNA 序列、長影片任務領先。然而 FlashAttention 等優化讓 Transformer 仍是工程主流，2025 年趨勢是「混合架構」(Hybrid Mamba-Transformer)。
Mamba vs Transformer · O(N²) vs O(N) — 為何序列愈長 Mamba 愈香
紅 = Transformer N×N 矩陣（每 token 互看，平方爆炸）· 綠 = Mamba 線性鏈式狀態傳遞（h 是歷史摘要） — 短序列 Transformer 強，超長序列 Mamba 才是唯一解
📌 補充 · Diffusion Transformer (DiT) — Sora 的「擴散 + Transformer」雙劍合璧 2024 年 2 月 OpenAI Sora 震撼問世，其架構稱為 DiT (Diffusion Transformer) — 把擴散模型原本的 U-Net 去噪器換成 Transformer。Sora 將影片切成「時空塊」(spacetime patches)，類似 ViT 處理影像的 patches；這讓影片生成首度具備 Transformer 的「scaling law」 — 模型愈大、資料愈多，生成品質持續提升。DiT 從此成為 Stable Diffusion 3、Flux、Hunyuan Video 等新一代生成模型的共同骨幹。
DiT 架構 · 把 Diffusion 的 U-Net 去噪器換成 Transformer，影片首次有了 Scaling Law
黃 = U-Net Diffusion（CNN 編解碼）· 紫 = DiT（時空塊 → Transformer） — 同樣的「逐步去噪」核心，換了去噪器引擎，影片生成從此進入 LLM 一樣的「越大越強」時代

⑤ 生成式模型 — GAN / VAE / Diffusion

模型原理特性
GAN生成器 vs 判別器對抗訓練；G 產生偽樣本、D 判斷真偽圖像合成（StyleGAN）；訓練不穩定、易模式崩潰
VAE變分自編碼器；學習資料的潛在分佈，從噪聲重建潛在空間結構好；適合異常偵測、語音重建；生成相對模糊

三大生成模型對比 · 為什麼 Diffusion 從 2022 起一統天下
紅 = GAN（一步生但訓練亂）· 黃 = VAE（一步生但模糊）· 紫 = Diffusion（千步生但穩+高質）— Diffusion 用「化難為易」的數學魔法贏了 2022+ 的生成戰爭

E · 主流深度學習框架

2026 年三大框架已分出明確格局：PyTorch 統治研究與快速原型（NeurIPS 2024 論文 75% 用 PyTorch），TensorFlow 守住企業生產部署（Google、Uber、Airbnb），JAX 是 Google DeepMind 內部主力（Gemini、AlphaFold 皆用之，擅長大規模 TPU 訓練）。Keras 3 則打破壁壘 — 一份模型程式碼可在三家後端任意切換。Hugging Face 是貫穿三者的「開源 AI GitHub」，提供 100 萬+ 預訓練模型、一行 `from_pretrained()` 即可用。2025 年新趨勢是推理框架分離：訓練用 PyTorch、部署用 vLLM / TensorRT-LLM / SGLang，差距可達數十倍 token 吞吐量。

遷移學習與領域適應（PDF 外延伸 · 業界實務 + 考古題重點）

為什麼一個在英文資料上 F1=0.91 的模型，搬到西班牙文資料上 F1 只剩 0.58？這不是模型訓練不夠久 — 是 領域漂移（Domain Shift）。實務上多數模型部署到新場景都會遇到這個問題：訓練資料分佈跟測試分佈不一致，模型「水土不服」。遷移學習（Transfer Learning）是業界標準解法，也是 iPAS 考試的常見題型（114 年第 30 題、第 45 題皆考此主題）。本節從問題本質開始，逐步介紹三種主流解法 + 凍結 layer 實作技巧。

📌 補充 · 凍結 layer 的 3 種 PyTorch 寫法（考古題 45 對應） ① requires_grad（最常用）：逐層設 param.requires_grad = False — 標記為「不計算梯度」，optimizer 自動跳過。
② optimizer param_groups：建 optimizer 時只傳要訓練的參數，optimizer = Adam(model.classifier.parameters(), lr=1e-4)。
③ 設置 layer 為 eval()：對 BatchNorm / Dropout 特別重要 — 凍結 conv layer 後務必 model.conv1.eval()，否則 BN 統計值會被新資料污染。
⚠ 常見錯誤：只用 ① 但忘了 ③ — 結果 BN running mean/var 被更新、效能不穩。正確做法：requires_grad=False + .eval() 都要做。詳細程式碼見 §7.5 遷移學習實作。

LLM 應用三大模式：Prompt Engineering · RAG · Fine-tuning（PDF 外延伸 · 業界實務）

2023 年 ChatGPT 引爆 LLM 浪潮後，AI 應用規劃師面對的核心問題不再是「選哪家模型？」（OpenAI / Anthropic / Google / Meta 都很強），而是「用什麼模式跟 LLM 互動？」。三大主流模式：Prompt Engineering（零訓練、純設計）→ RAG（檢索增強生成）（外掛私域知識防幻覺）→ Fine-tuning（改變模型本身行為）。三者不是替代關係、是互補的工程層次。本節雖 PDF / 考古題都沒考，但是 2024-2026 業界基本功 — 不會這三個模式就上不了戰場。

機器學習應用流程與技術

建模的完整流程：數據準備 → 特徵工程 → 模型選擇 → 訓練 → 評估 → 調整優化。這是把演算法落地的關鍵工程實踐。

數據準備與特徵工程

從 §3-§4 的理論進入實務戰場 — 機器學習真正的勝負，70% 在這一節決定。Andrew Ng 在 2021 年發起的「Data-centric AI」運動有句名言：「You don't improve your model by tweaking hyperparameters. You improve it by improving your data.」這不是學界口號 — Tesla 自駕系統團隊（Karpathy）、Google 翻譯團隊、OpenAI GPT-4 訓練團隊都印證了同一件事：當模型架構進入紅海後，資料工程成為唯一的差異化武器。本節從資料清理 → 特徵選擇 → 特徵轉換 → 資料增強，建構完整的「資料工匠」工具箱。

資料品質 > 演算法 · ML 鐵則 ML 流程中「資料品質與特徵表達」往往比演算法本身更決定模型表現。2009 年 Netflix Prize — Netflix 拿出 $100 萬美元獎金徵求改進電影推薦演算法的團隊，三年戰況最後決勝的不是「誰用了最新的矩陣分解」，而是誰把使用者「評分時的心情、評分當下的時段、最近看過的電影、季節變化」這些『資料側』特徵抓得最精準。冠軍團隊 BellKor 用的演算法跟其他人沒差多少 — 但他們的特徵工程讓他們 RMSE 領先 10%。「數據準備與特徵工程是模型效能的基礎建設」這句話的真正意義：演算法是地基上面的建築，資料是地基本身。

  • SVD：矩陣分解，文字分析（LSA）、推薦系統

黃 PCA · 紅 LDA · 紫 t-SNE — 同樣三類資料，PCA 找「展最開」、LDA 找「分最開」、t-SNE 保「鄰近關係」 — 用錯就是用錯

③ 類別資料編碼

遇到「台北 / 高雄 / 台中」這種文字類別，模型不會讀字 — 必須轉成數字。但「轉成 1, 2, 3」會引入嚴重副作用：模型誤以為「台中 = 3 比台北 = 1 大三倍」，導致預測荒謬。這就是為何要區分三種編碼策略，選錯編碼比選錯模型還傷。

④ 時間與週期性編碼

時間是 ML 中最容易處理錯誤的特徵 — 「星期幾」如果直接用 1-7 表達，模型會誤以為「星期日（7）距離星期一（1）= 6 步」，但實際上它們相差只有 1 天！這就是週期性編碼要解決的問題：用 sin/cos 雙軸把時間「卷成圓圈」，讓模型理解「12 月 31 日跟 1 月 1 日」其實是緊鄰的。這個技巧在能源預測、交通流量、零售銷售預測中是標配。

  • 結構化拆解：把時間戳分解為「年、月、日、星期幾、時段」
  • 週期性編碼：用 sin / cos 三角函數轉換保留週期結構（一週循環、月份循環）
時間週期編碼 · 為什麼「星期幾 = 1~7」會讓模型誤判 6 倍距離
紅 = 線性編碼（誤判 6 倍距離） · 綠 = sin/cos 圓圈編碼（任意相鄰兩天等距） — 模型理解「時間是循環的、不是直線」

D · 資料增強策略

當少數類別樣本太少（詐欺交易、罕見疾病、設備故障），模型會自動「偷懶」 — 全部預測為多數類別，準確率看起來 99%，但實際上一個正例都沒抓到。SMOTE（Synthetic Minority Over-sampling Technique，2002 Chawla 發表，引用 26000+ 次）解法很聰明：不複製、而是合成。在少數類別樣本之間「畫線、取中點」，創造出從未存在過、但合理的新樣本，讓模型看到更多正例的「樣貌」。

類型方法應用

文字同義字替換、隨機插入/刪除/交換情感分析、主題分類
時序添加噪聲、Scaling、Jittering、視窗裁切設備故障、股市趨勢

紅 = 原始少數類 · 紫 = SMOTE 合成 · 灰 = 多數類 — SMOTE 不是複製、是「在原始少數類樣本連線上取點」生成從未存在過但合理的新樣本

E · 特徵工程的策略思考

特徵工程不只是技術操作 — 它是「資料理解 + 領域知識 + 建模目標」的整合決策。PDF 強調這是一個「有目的、有規劃、需要深思熟慮的決策過程」，並提出 4 大思考維度：任務類型 · 模型性質 · 隱含結構 · 時序關聯。每個維度都對應一組可立刻動手的實務技巧。

特徵工程策略思考 · 4 大維度架構圖（PDF §5.1 第 6 點原文視覺化）
中心 = 特徵工程策略思考 · 4 個顏色 = 4 個思考維度（任務 / 模型 / 結構 / 時序） — 設計新特徵時，依序問自己這 4 個維度即可
「策略思考」的本質 · 不是套招、是「對話」 特徵工程不是查清單套技巧，而是一連串「跟資料與業務對話」的過程。每設計一個新特徵，問自己 4 個問題 — 「這對分類 / 迴歸有幫助嗎？」「這個模型吃得下這個格式嗎？」「這個欄位有沒有隱藏的組合或群體訊息？」「這個任務有時間維度嗎？」。能在 4 個維度都答得出來的特徵，才是值得加入訓練集的特徵。
常考點：缺失值與特徵編碼選擇 題目情境問「處理缺失值的方法？」 → 插補法（Imputation）。問「對無序類別變數（如城市）編碼？」 → One-hot Encoding（用 Label Encoding 模型會誤解為大小關係）。問「特徵標準化目的？」 → 把不同尺度特徵縮放至相近範圍，利於模型學習。
📌 補充 · 互動 / 交互特徵設計（考古題 20 對應） 原始特徵不夠用？業務上常常需要組合既有特徵生出新特徵 — 這就是「互動特徵（Interaction Features）」。三種主流形式：

① 乘積（最常見）：客單價 × 購買頻率 = LTV · 身高 × 體重 = BMI 雛形 · 日均瀏覽 × 留存天數 = 沉浸指數 — 反映「強度 × 頻率」的真實意義
② 多項式特徵（Polynomial）：x², x×y, y² — sklearn 的 PolynomialFeatures(degree=2) 自動生 — 線性模型抓非線性關係的關鍵
③ 分段組合（Binning + Cross）：把連續變數分箱後做笛卡爾積 — 例：「年齡段 × 收入段」生 9 個離散桶，比原始連續變數對樹模型更友善
實務鐵則：① 先靠領域知識手做（單價×數量、距離×時間）② 再用 sklearn PolynomialFeatures 自動補 ③ 用 L1 正則化淘汰無用組合（避免維度爆炸）。

E · 偏差-變異權衡（Bias-Variance Tradeoff）

Bias-Variance Tradeoff 是傳統 ML 教科書的金科玉律 — 模型複雜度愈高，偏差降但變異升，總誤差呈 U 形曲線。但 2019 年 OpenAI / Belkin 等人發現一個顛覆性現象：當模型「過度參數化」（如 GPT-4 1.8 兆參數）時，誤差在 U 形之後會再次下降 — 這就是 Double Descent（雙下降）。本節先講傳統 U 形，再揭示 LLM 時代的新景觀。

Bias-Variance Tradeoff · 從傳統 U 形到 2019 Double Descent 新景觀
左 = 傳統 U 形（小-中模型，仍然主流）· 右 = Double Descent（大模型 / LLM 時代）— 同一條曲線、不同尺度，是兩個世界
概念說明
偏差 Bias模型對資料結構的擬合能力不足 → 無法捕捉主要趨勢 → 欠擬合（Underfitting）
變異 Variance模型對訓練資料的細微差異或雜訊過於敏感 → 不同資料集表現差異大 → 過擬合（Overfitting）
取捨低複雜度模型（線性）= 高偏差 + 低變異；高複雜度模型（深度 NN）= 低偏差 + 高變異
實務原則：找到 Sweet Spot 模型選擇 = 在偏差與變異間取得平衡。一般做法：① 從簡單模型開始（線性、邏輯迴歸）建基線；② 如效能不夠，逐步增加複雜度（樹模型、集成、深度學習）；③ 用交叉驗證監控變異；④ 用正則化抑制過擬合。

F · AutoML 與神經架構搜尋（PDF 外延伸 · 2024-2026 業界實務）

前五節討論的是「人工」模型選擇 — 工程師根據經驗判斷。但 2024-2026 業界已進入「機器選模型」時代：AutoML（自動機器學習）+ NAS（神經架構搜尋）讓非 ML 專家也能訓出 SOTA 模型，並把模型選型時間從「月級」壓到「天級」甚至「小時級」。本節介紹三大主流工具與適用場景，是 iPAS 應用規劃師應該認知的業界現況，雖然不在 PDF 考綱中，但與你未來實際接觸的 AI 專案高度相關。

📌 補充 · 三大 AutoML 工具與適用場景 Hugging Face AutoTrain（2024 升級）：專注 NLP / Vision 預訓練模型微調 — 上傳資料 + 選任務類型，自動挑模型、調超參數、輸出可部署 endpoint。適合：業務團隊快速 PoC、無 ML 專家的中小企業。

iPAS 應用規劃師啟示：未來 AI 專案不再從「挑模型」開始 — 而是從「定義目標 + 資源約束」開始，讓 AutoML / NAS 自動找解。你的核心價值轉移到：業務理解、需求釐清、模型治理（fairness、explainability、deployment）。

③ 解釋性與審計困難 — AutoML 黑盒搜尋的結果，比人工選型更難向法規 / 業務方解釋「為何選這個模型」。

紅點 = 真實病患（20 人） · 藍點 = 真實健康（80 人）· 綠虛線 = 低閾值（漏抓代價高的場景） · 紫虛線 = 高閾值（誤報代價高的場景）— 同一個模型、不同閾值、完全不同的臨床決策

五大核心指標

指標公式解讀適用情境
Accuracy 準確率(TP + TN) / 全部整體預測正確比例類別均衡；不平衡時誤導
Precision 精確率TP / (TP + FP)預測為正中真正為正的比例誤報代價高：垃圾郵件、醫療誤診
Recall 召回率TP / (TP + FN)真實正中被抓到的比例漏報代價高：癌症偵測、詐騙偵測

ROC-AUCROC 曲線下的面積所有閾值下的整體區分能力；越接近 1 越好需綜觀整體預測能力
Accuracy 在類別不平衡時的陷阱（考古題 19 對應） 若 95% 樣本為負類，即使模型全預測為負，準確率仍 95% — 但實際模型沒抓到任何正類！詐欺偵測、罕病診斷等場景絕不能只看 Accuracy，必須看 Precision、Recall、F1、AUC。

具體數字感受：100 萬筆交易中 1500 筆是詐欺（不平衡 1:670）— 模型「全部預測為正常」可得 99.85% Accuracy，但 Recall = 0%（一個詐欺都沒抓到）。關鍵指標選擇：① 看少數類別 → Recall + F1；② 看模型整體區分力（與閾值無關）→ ROC-AUC；③ 嚴重不平衡（< 1%）→ PR-AUC 比 ROC-AUC 更敏感（已詳述於 §5.3 B ROC vs PR）。
ROC vs PR 曲線 · 兩條曲線、不同情境（不平衡資料的關鍵分歧點）
左 ROC = 看整體區分力（穩定）· 右 PR = 看不平衡資料的正類表現（敏感）— 2024 新共識：兩條都看才完整，不要再爭「哪條比較好」

D · 交叉驗證（Cross-Validation）

為什麼需要交叉驗證？— 只切一次 train / val 太冒險。如果剛好那一次的驗證集包含了「容易的樣本」，模型分數會虛高；包含「難的樣本」會虛低。K-fold 把資料切 K 份輪流當 val，取平均分數，讓評估更穩定可靠。本節用一張圖解視覺化 K-fold 的核心機制，然後介紹四種變體。

5-Fold Cross-Validation 視覺化 · 5 個 fold 輪流當驗證集
紅 = 驗證集（每輪輪流）· 綠 = 訓練集 · 紫 = 各輪分數 — 5 輪結束後取平均 0.888 ± 0.018 是穩定可信的效能估計

K-fold Cross-Validation

原理：把資料分為 K 個不重疊子集（folds），輪流以其中 1 折作為驗證集、其餘 K-1 折作訓練。重複 K 次取平均。

特點：減少資料切分偏差；常見 K=5 或 10；計算成本為 K 倍。

適用：中大型資料集；模型選擇與效能比較的標準方法。

Stratified K-fold 分層交叉驗證

原理：K-fold 的改良版 — 每一折中各類別比例與整體資料集相符（依類別分層抽樣）。

適用：類別分佈不均的分類問題（詐欺、疾病、異常事件）；是分類任務的預設驗證方式。

LOOCV — Leave-One-Out

原理：K-fold 極端版本，K = 樣本數 n。每次留 1 筆驗證、其餘 n-1 訓練，總共 n 次。

適用：樣本極小但珍貴（臨床研究、稀有病）。計算成本極高。

Repeated K-fold 重複交叉驗證

原理：在 K-fold 基礎上多次隨機重劃（如 10-fold × 5 次），取多組結果計算平均與變異。

適用：學術研究需報告標準差與置信區間、產業建模需重現性高。

4 種 CV 變體並列對比 · 從「基本 K-fold」延伸出的差異與適用場景
藍 = K-fold（標準）· 綠 = Stratified（分類預設）· 紅 = LOOCV（極端）· 紫 = Repeated（學術）— 每個變體用專屬色、底部 callout 文字色彩直接對應，視覺一致

策略原理

模型調整與優化

即使完成初步訓練，實務上仍常面臨效能不穩、推論延遲、過擬合、資源消耗過高等問題。本節聚焦模型建構後的優化行動：超參數調校、正則化、資料策略、模型壓縮。

A · 超參數調校

超參數（Hyperparameters）是需開發者手動指定的參數，不由資料自動學習得出。對訓練穩定性、收斂速度、最終效能影響深遠。實務上，挑超參數有兩步：① 哪些超參數要調（下方 6 大關鍵）；② 怎麼有效率地搜尋最佳組合（先做這 3 法對比）。

超參數搜尋方法 · Grid vs Random vs Bayesian

超參數搜尋三法 · 在 2D 空間（學習率 × 批次大小）中的搜尋軌跡對比
黃 = Grid 規則格點（窮舉 25 次）· 綠 = Random 隨機散點（12 次命中 2 次）· 紫 = Bayesian 智能搜尋（5 次直接命中）— 效率差距 5 倍以上

六大關鍵超參數

① 學習率 Learning Rate

角色：控制每次反向傳播後參數更新的幅度。

影響：過高 → 跳過最小點、震盪甚至發散；過低 → 收斂慢、陷入局部極小。

實務：初期較大、後期較小（搭配 LR Scheduler 退火策略）。

② 批次大小 Batch Size

影響梯度估計穩定性、模型更新頻率、資源使用。

小批次：梯度波動大、易跳脫局部最小；訓練不穩。

大批次：梯度穩定、加速收斂；但可能降低泛化、陷入局部最小。

耦合：Batch Size 與學習率高度相關，放大 Batch 通常需提高學習率。

③ 網路深度與寬度

深度（層數）：提升非線性表達能力。寬度（每層神經元數）：豐富特徵表示空間。

過深：易梯度消失（需殘差連接、BatchNorm 緩解）。過寬：模型容量過大易過擬合。

過淺/過窄：表達能力不足，欠擬合。

④ 激活函數

激活函數適用
Softmax輸出層 — 多類別單選（類別互斥）；輸出機率分佈，總和為 1
Sigmoid輸出層 — 二元分類；每個輸出獨立表示機率
ReLU隱藏層主力 — 解決梯度消失、計算快
Tanh隱藏層 — 中心對稱（-1 到 1）

⑤ 優化器 Optimizer

優化器特性 / 適用
SGD最基本；簡單、記憶體低；對學習率敏感
MomentumSGD + 動量；減少震盪、加速收斂
Adagrad自適應學習率；適合稀疏特徵（NLP）；但學習率持續衰減
AdamMomentum + RMSprop；收斂快穩；萬用優化器；幾乎所有深度學習

⑥ 正則化係數 λ

λ 大：強化限制，權重壓縮更小，模型簡單但可能欠擬合。

λ 小：放鬆限制，權重可變大，模型靈活但易過擬合。

調整：需透過交叉驗證選擇。

6 大超參數影響圖譜 · 過大 vs 過小的對比效應
中心 hub「訓練」+ 6 方向輻射 6 大超參數 — 紅黃綠橘藍紫 6 種色系對應各超參數的「影響軸」與「過大過小的代價」
超參數調校的「黃金順序」 不是 6 個一起調 — 先固定其他、單獨調最重要的： ① 學習率（最重要、影響最大）→ ② 批次大小（耦合學習率）→ ③ 優化器（Adam 多半就夠）→ ④ 正則化係數 λ（防過擬合）→ ⑤ 網路深度與寬度（容量）→ ⑥ 激活函數（多半 ReLU 就好）。 這個順序也是「最影響表現 → 最不影響」的排序 — 對應前面 SVG 講的 Bayesian Optimization 也是先聚焦在重要超參數的方向。

B · 正則化技術詳述

技術原理特性
L1 正則化 LassoLoss + λ Σ|θᵢ|（絕對值總和）促使部分權重為零 — 自動特徵選擇；高維資料適用
L2 正則化 RidgeLoss + λ Σθᵢ²（平方和）限制權重大小但不為零；解決多重共線性；平滑模型
Elastic NetL1 + L2 結合同時稀疏 + 穩定；多重共線性 + 特徵選擇
Dropout訓練時隨機關閉部分神經元（保留率 p 常為 0.5）讓模型學多種子網路的特徵；測試時不丟棄但縮放權重
Early Stopping驗證集不再改善時提前停止用 patience 參數控制；無需修改架構
補充：Lasso 與 Ridge 的命名典故 Lasso = Least Absolute Shrinkage and Selection Operator — 絕對值（Absolute）+ 縮小（Shrinkage）+ 自動選擇（Selection）。
Ridge：統計中多重共線性下，普通最小平方（OLS）估計不穩；加 L2 後解會沿著「脊線（Ridge）」分佈，幾何上呈橢圓形範圍 — 故名 Ridge。
L1 vs L2 正則化 · 為什麼 L1 自動「歸零特徵」、L2 只是「縮小」？
紫色等高線 = 損失函數（離 OLS 解愈近愈低）· 紅色鑽石 = L1 限制 · 黃色圓 = L2 限制 — 切點位置決定為何 L1 自動稀疏、L2 只是平滑
Dropout 訓練 vs 測試 · 一個網路 = 多個「子網路」的隱性集成
紅 = 訓練時隨機關閉的神經元 · 灰 X = 被關閉的神經元 · 綠 = 測試時全部恢復 — Dropout = 「強制團隊每個成員都得獨立有用」

D · 模型壓縮與加速

TECH 01
知識蒸餾
Knowledge Distillation
把大型「教師模型」的知識傳遞給較小的「學生模型」。學生體積小、推論快、效能接近教師。
應用：邊緣裝置部署、降低延遲。
TECH 02
模型剪枝 Pruning
結構化（移除整個神經元/層）：易硬體加速。
非結構化（移除單一權重）：稀疏化但硬體支援差。
過度剪枝會降準確度。
TECH 03
量化 Quantization
高精度（FP32）→ 低精度（INT8）。大幅降低儲存與運算負擔。
QAT 量化感知訓練讓模型適應量化誤差。Edge TPU、NPU 已最佳化。
TECH 04
混合精度訓練
Mixed Precision
FP16 + FP32 同時使用。加速訓練、減少記憶體，允許更大批次或更深模型。NVIDIA Tensor Cores 已最佳化。
知識蒸餾 Teacher → Student · 為何小模型也能跑出 95% 大模型效果
紅 Teacher 大模型 → 黃 軟標籤（帶相似性資訊）→ 綠 Student 小模型 → 紫 部署到邊緣 — 「大模型教小模型」是 LLM 邊緣化的關鍵技術
量化技術 · FP32 → INT8 / GPTQ / AWQ 的精度損失 vs 模型大小權衡
紅虛線 = 原生量化（INT4 後崩潰）· 黃 = GPTQ · 綠 = AWQ（2024 Best）— 同樣 INT4 位元，AWQ 把「準確率懸崖」變成「平滑下降」
常考點：超參數與優化技術辨識
  • 超參數 vs 模型參數：超參數需手動設定（學習率、Batch Size、層數）；模型參數由訓練自動學習（權重、偏差）
  • L1（Lasso）→ 自動特徵選擇；L2（Ridge）→ 平滑權重不為零
  • Dropout 是訓練時隨機關閉神經元，測試時恢復
  • 知識蒸餾把大模型壓成小模型；量化用低精度數值；剪枝移除不重要權重
  • 不平衡資料用 SMOTE 合成或類別加權

A · 三大數據隱私風險類型

2024-2026 全球個資與 AI 治理進入「嚴格執行期」 — GDPR 罰款已突破累計 €40 億、台灣個資法修訂、EU AI Act 開始分階段生效、美國各州 CCPA 升級。資料外洩或濫用不再是 IT 問題，是法務、商譽、刑事責任三重風險。本節從 3 大風險出發，建構完整的隱私治理思維。

基礎匿名化技術（五種）

技術作法限制
遮蔽 Masking用符號或虛構數據替換敏感欄位（王○○、A12******89）僅隱藏部分資訊，易結合其他資料還原
雜湊 Hashing單向雜湊（SHA-256）生成不可逆字串原始範圍小時易受彩虹表攻擊；無法數值分析
泛化 Generalization降低精度（1987-03-12 → 1980 年代；信義區 → 台北市）精度損失影響分析準確度
分桶 Bucketing連續數值轉區間（58,000 → 50K-60K；32 歲 → 30-39 歲）分桶設計過細仍可能暴露個體

進階隱私強化技術（PETs）

K-Anonymity / L-Diversity / T-Closeness

K-匿名：確保每筆紀錄至少與 K-1 筆其他紀錄在準識別欄位相同 — 個體「藏在」群中。

L-多樣性：在 K-匿名基礎上要求每個分組內敏感欄位（如疾病）有至少 L 種不同值 — 防止同質性攻擊。

T-接近性：進一步要求群組內敏感欄位的分佈與全體資料集相近 — 防止分佈偏差推測。

K-匿名 → L-多樣 → T-接近 · 三階段防護演化（病歷資料案例）
同一份病歷資料的三階段演化：紅 = 原始（高風險） → 黃 = K-匿名（防直接識別但有殘存風險） → 紫 = L-多樣（防同質但仍有分佈偏斜） → 綠 = T-接近（全防）。關鍵：三者層層加強，但會逐步降低資料效用（precision loss）。

聯邦學習 Federated Learning

模型在各客戶端（裝置或機構）本地訓練，只傳模型參數更新（梯度），原始資料不外流。

應用：醫療聯盟跨院建模、行動裝置鍵盤預測（Gboard）。

限制：通訊成本高、參數逆向推導風險。

聯邦學習架構 · 「資料不出端、只傳模型參數」的 5 步驟循環
綠色虛線 = 只傳模型梯度（資料留在端上）· 黑色實線 = 下發全域模型 — 經 5 步循環不斷迭代直到全域模型收斂。關鍵差別：傳統 ML 把資料上傳到中央訓練，聯邦學習把訓練下放到資料所在端。
真實案例 · Google Gboard 鍵盤預測（聯邦學習史上最大規模部署） 全球數十億支 Android 手機 = 數十億個 client。① 每支手機本地學使用者打字模式 → ② 上傳前用 HE 加密梯度 → ③ Secure Aggregation 算總和 → ④ 加 DP 噪聲後彙整成新模型。結果：Google 永遠看不到你打了什麼，但全球輸入法仍持續變聰明。Apple PCC（2024）、Meta NLLB 翻譯模型、台灣健保聯邦學習都是同樣架構。

同態加密 Homomorphic Encryption

在加密資料上直接運算（加、乘），解密後結果與明文運算一致。

優勢：資料全程加密、密碼學強保證。限制：計算效能低，特別是 FHE 全同態加密需高效能硬體。

同態加密工作流程 · 「在密文上直接運算」雲端不解密就能算
綠色虛框 = 客戶端信任區（持有金鑰）· 紅色虛框 = 雲端不信任區（看不到原始資料）。關鍵：雲端只看到密文 E(x)，卻能算出 E(x²)、E(x+y) 等運算結果 — 解密後就是 x²，全程資料零洩漏。
📌 補充 · PETs 多技術組合方案（考古題 37 對應） 業界實務不會「只用一個 PETs」 — 而是多種技術層疊組合達到更強的隱私 + 完整性保證。考古題常考「同時用 X + Y + Z 達成什麼？」

常見 4 法組合（聯邦學習場景）：
層技術解決什麼問題
傳輸層同態加密（HE）傳輸過程中參數加密、伺服器看不到原始梯度
身份層數位簽章 + 雜湊（SHA-256）確認參數來自合法 client、防偽冒
計算層安全多方計算（MPC）多方共同算總和、無人知道個別值
結果層差分隱私（DP）最終彙總前加噪、防回推單一 client
真實案例 · Google Gboard 鍵盤預測：① 每支手機本地訓練（聯邦學習）→ ② 上傳前用 HE 加密 → ③ 用 Secure Aggregation 算總和 → ④ 加 DP 噪聲。結果：Google 永遠不會看到你打了什麼字，但全球模型仍能改進。

iPAS 考點：看到「確保資料完整性 + 隱私 + 不可偽造」三項全包的題目，答案常是「HE + 數位簽章 + MPC + DP」組合，不是單一技術。
📌 補充 · 差分隱私（Differential Privacy） · 2024-2025 LLM 訓練的標配 差分隱私（DP） 是當前最受重視的 PETs — 在資料統計或模型訓練時加入數學上可證明的噪聲，讓「加入 / 移除單一筆資料」對結果的影響在可控範圍內。簡單說：單一個人的資料無法被推斷出來，但整體統計仍可用。
2024-2025 真實應用：
• Google VaultGemma（2024）：首個生產級 DP LLM，用 DP-SGD 訓練、無法洩漏訓練資料中的個人資訊
• Apple Private Cloud Compute（2024）：iOS 18 雲端推論加入 DP 機制，端到端可審計
• 美國人口普查局（2020 普查）：首次用 DP 發布資料，防止「重識別攻擊」
核心參數 ε（隱私預算）：ε 小 = 隱私強但效用低；ε 大 = 效用高但隱私弱。典型值 ε=1-3。這是 2024-2026 隱私工程的關鍵指標。
差分隱私機制 · 「加噪 → 防個體識別」與 ε 預算的權衡
三步驟機制：真實查詢 → 加噪（Laplace 或 Gaussian） → 含噪結果（每次不同）。ε 軸：噪聲量 ∝ 1/ε — ε 越小噪聲越大隱私越強、ε 越大噪聲越小效用越高。業界主流 ε=1-3。Central DP（伺服器加噪、高效用）vs Local DP（client 加噪、零信任）為兩大實作策略。
PETs 五大技術權衡圖 · 「隱私強度 × 計算成本 × 應用場景」三維選型
紅 = 弱保護（遮蔽 / 雜湊）· 黃 = 中度（泛化 / K-匿名）· 紫 = 強保護（DP / 聯邦 / 同態）— 從左下到右上：保護越強，計算也越貴，業界依資料敏感度選型

B · 完整範例 · 向量點積與餘弦相似度

考古題第 40 題：「計算兩向量 a = [1, 2, 3]、b = [4, 5, 6] 的點積」答案 = 1·4 + 2·5 + 3·6 = 32。延伸到 ML 實務 — 計算餘弦相似度用於文字 embedding 比對、推薦系統：

📄 vector_ops.py · 點積 + 餘弦相似度
import numpy as np

# 兩個向量
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

# 點積三種等價寫法
dot1 = np.dot(a, b)         # 32 (傳統)
dot2 = a @ b                 # 32 (Python 3.5+ 推薦)
dot3 = np.sum(a * b)         # 32 (手算等價)

# 餘弦相似度 = 點積 / (norm_a × norm_b)
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(cosine_sim(a, b))   # 0.9746 (越接近 1 越相似)

C · 矩陣乘法的 shape 規則（最常踩雷）

矩陣 A (m×n) × B (p×q) 只有當 n == p 才能乘，結果 shape 為 (m, q)。考題與面試最愛考「給你 A、B 兩個矩陣，能不能相乘？結果 shape 多少？」

📄 matrix_mul.py · 矩陣乘法 shape 推導
import numpy as np

A = np.random.rand(3, 4)   # shape (3, 4)
B = np.random.rand(4, 2)   # shape (4, 2)

C = A @ B                   # OK: (3,4)·(4,2) = (3,2) ✓
print(C.shape)            # (3, 2)

# 錯誤示範:
# D = B @ A  # 報錯: (4,2) @ (3,4) → 2 ≠ 3

# 解法: 用 transpose 對齊
D = B.T @ A.T               # (2,4) @ (4,3) = (2,3) ✓
🎯 考古題對應 · §7.1 第 40 題：「向量點積」直接考 np.dot。第 42-44 題：VGG16 結構推導需要懂 Conv2d 輸出 shape 公式（見 §7.4）。第 46 題：PCA 程式碼依賴 NumPy 線代運算。
⚠ NumPy 三大陷阱 ① shape 不對齊：「ValueError: shapes (3,4) and (3,4) not aligned」 — 用 A.shape, B.shape 印出檢查、再決定要不要 transpose。
② 整數 vs 浮點：np.array([1,2,3]) 是 int64、np.array([1.,2.,3.]) 是 float64 — 計算機率 / 距離時用 float。

from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split, cross_val_score, StratifiedKFold
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, classification_report

# ① 載入資料
X, y = load_iris(return_X_y=True)

# ② 切分（stratify 保持類別比例 — 不平衡資料必做）
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# ③ 標準化 — fit 只用訓練集（防資料洩漏！）
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)   # transform 不 fit

# ④ 訓練
model = LogisticRegression(max_iter=200)
model.fit(X_train_s, y_train)

# ⑤ 評估
y_pred = model.predict(X_test_s)
print(accuracy_score(y_test, y_pred))   # 0.97

# 進階: 5-fold Stratified CV
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring='accuracy')
print(f'5-fold CV: {scores.mean():.3f} ± {scores.std():.3f}')

C · Pipeline 防止資料洩漏（考題 47 對應）

考古題第 47 題考「KNN 交叉驗證正確寫法」— 重點是 標準化必須在 CV 內進行，否則「先標準化全部資料 → 再切 CV」會洩漏測試集資訊。Pipeline 是標準解法：

📄 pipeline_cv.py · 正確的 CV 流程
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import cross_val_score

# ❌ 錯誤: 先標準化全部資料、再 CV (資料洩漏!)
# X_scaled = StandardScaler().fit_transform(X)   # 用了測試集統計值
# cross_val_score(KNeighborsClassifier(), X_scaled, y, cv=5)

# ✓ 正確: Pipeline 確保每折獨立標準化
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('knn', KNeighborsClassifier(n_neighbors=5))
])
scores = cross_val_score(pipe, X, y, cv=5)   # 每折只用該折訓練集 fit scaler
🎯 考古題對應 · §7.2 第 1 題：交叉驗證提升泛化能力 — 答案核心是 cross_val_score(cv=5)。第 9 題：特徵縮放 — StandardScaler。第 47 題：KNN 交叉驗證正確寫法 — 必用 Pipeline。第 48 題（多選）：標準化的多重效果（梯度穩、收斂快、距離可比）。
⚠ 三大資料洩漏陷阱 ① 對全部資料先標準化、再切 CV：scaler 看到測試集 → 評估虛高。修法：用 Pipeline 包起來。
② Target Encoding 直接用全資料平均：洩漏 label 資訊。修法：用 out-of-fold encoding（每折分開算）。
③ 特徵工程時用未來資訊：時序預測時 df.mean() 包含未來。修法：用 expanding window 或 rolling 統計。

B · Conv2d 輸出 shape 公式（VGG16 推導必備）

📐 Conv2d 輸出 shape 公式（必背） 給定輸入 (N, C_in, H, W)，經 Conv2d(C_in, C_out, kernel=K, stride=S, padding=P) 後：

輸出 H_out = ⌊(H + 2P − K) / S⌋ + 1
輸出 shape = (N, C_out, H_out, W_out)

常見情況：kernel=3, stride=1, padding=1 → H 不變（same padding）；kernel=3, stride=2, padding=1 → H/2。
VGG16 例：輸入 (1, 3, 224, 224) → 13 個 Conv (3, S=1, P=1，不縮) + 5 個 MaxPool (2, S=2，每次 /2) → 最終 (1, 512, 7, 7)。

import torch
import torch.nn as nn
import torch.optim as optim

# ① 定義模型 (考題 49: 隱藏層維度推導)
class MLP(nn.Module):
    def __init__(self, in_dim=100, hidden=64, out_dim=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden),   # 100 → 64
            nn.ReLU(),                     # 隱藏層用 ReLU
            nn.Dropout(0.3),               # Dropout 防過擬合 (考題 39)
            nn.Linear(hidden, out_dim)    # 64 → 2 (二分類)
        )
    def forward(self, x):
        return self.net(x)

model = MLP()
criterion = nn.CrossEntropyLoss()   # 多分類用（自動含 softmax）
optimizer = optim.Adam(model.parameters(), lr=1e-3)

# ② 訓練迴圈
for epoch in range(10):
    model.train()                    # 訓練模式 (Dropout 開啟)
    for X_batch, y_batch in train_loader:

        logits = model(X_batch)         # 前向
        loss = criterion(logits, y_batch)
        loss.backward()                # 反向
        optimizer.step()                # 更新參數

# ③ 推論
model.eval()                        # 推論模式 (Dropout 關閉)
with torch.no_grad():               # 不計梯度 (省記憶體)
    logits = model(X_test)

A · 三種「凍結 layer」標準寫法

方法語法適用
① requires_grad = Falsefor p in model.parameters(): p.requires_grad = False最常用、最徹底
② optimizer 只傳要訓練的optim.Adam(model.classifier.parameters())不改 requires_grad、彈性高
③ 切換 eval 模式model.conv1.eval()BatchNorm/Dropout 必加（否則統計值會被更新）

B · 完整範例 · ResNet18 遷移學習做花卉分類

📄 transfer_learning.py · ImageNet 預訓練 → 花卉 5 類分類
import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import models

# ① 載入 ImageNet 預訓練 ResNet18
model = models.resnet18(weights='IMAGENET1K_V1')

# ② 凍結全部 conv layer (Feature Extraction 模式)
for param in model.parameters():
    param.requires_grad = False

# ③ 替換最後的 classifier (花卉 5 類)
#    替換後的 fc layer requires_grad 預設 True (要訓練)
num_features = model.fc.in_features    # 512 (ResNet18)
model.fc = nn.Linear(num_features, 5)   # 5 種花

# ④ 對 BatchNorm 切 eval 模式 (避免統計值漂移)
for m in model.modules():
    if isinstance(m, nn.BatchNorm2d):
        m.eval()                    # BN 不更新 running stats

# ⑤ Optimizer 只傳「需要梯度」的參數 (只有 fc layer)
optimizer = optim.Adam(
    filter(lambda p: p.requires_grad, model.parameters()),
    lr=1e-3
)

# 訓練 5 epochs (因為只訓 fc，幾分鐘內完成)
criterion = nn.CrossEntropyLoss()
for epoch in range(5):
    model.train()
    # 注意: model.train() 會把所有 layer 切 train 模式
    # 包括我們剛 .eval() 的 BN! 需重新切回:
    for m in model.modules():
        if isinstance(m, nn.BatchNorm2d):
            m.eval()                # 每 epoch 重切
    # ... 訓練迴圈

    {'params': model.layer4.parameters(), 'lr': 1e-5},
    {'params': model.fc.parameters(), 'lr': 1e-3}
])

# ② value_counts() vs groupby — 常被混淆
df["Platform"].value_counts().plot(kind="bar")
  # 各平台出現次數（不是銷售額）

# ③ 多重 groupby + agg
df.groupby(['Year', 'Genre'])['Global_Sales'].agg(['sum', 'mean', 'count'])
🎯 考古題對應 · 科二題 45 題 45：「統計每平台的全球銷售總額並以長條圖呈現」→ 標準答案 df.groupby("Platform")["Global_Sales"].sum().plot(kind="bar")。陷阱選項：.count() 是「筆數」、.mean() 是「平均」、value_counts() 是「該欄出現次數」 — 都不是「銷售總額」。

D · seaborn 視覺化 — barplot / countplot / lineplot

seaborn 是基於 matplotlib 的高階視覺化庫，與 pandas DataFrame 整合度高。應試常見三種：barplot（聚合長條）、countplot（自動計次）、lineplot（折線）。

📄 seaborn_basics.py · barplot / countplot / lineplot + melt（題 46 / 47 標準答案）
import seaborn as sns
import pandas as pd

# ① barplot — 聚合長條（指定 x, y）
sns.barplot(x="Name", y="NA_Sales",
            data=df.nlargest(5, "NA_Sales"))
  # 北美銷售前 5 名遊戲 ← 題 47 標準答案
  # .nlargest(N, col) 取該欄最大的 N 筆

# ② countplot — 自動計次（只指定 x）
sns.countplot(x="Platform", data=df)
  # 各平台出現次數

# ③ lineplot — 折線圖
sns.lineplot(x="Year", y="Global_Sales", data=df)

# ④ 多地區銷售比較 — 必須先 melt 轉長格式
df_long = pd.melt(df,
    value_vars=["NA_Sales", "EU_Sales", "JP_Sales", "Other_Sales"])
sns.barplot(x="variable", y="value",
            data=df_long, estimator=sum)
  # 四地區的銷售總額長條圖 ← 題 46 標準答案
  # pd.melt() 把寬格式（多欄）轉成長格式（一欄類別 + 一欄值）
⚠️ seaborn 常見陷阱 · wide format vs long format seaborn 的哲學是「tidy data（long format）」 — 每列一筆觀測、每欄一個變數。如果你的 DataFrame 是「寬格式」（如 NA_Sales、EU_Sales、JP_Sales 各一欄），seaborn 不能直接畫多區域比較圖，要先用 pd.melt() 轉成「長格式」（variable + value 兩欄）。
其他踩雷：① countplot 只能傳 x 或 y 一個（兩個都傳會報錯）；② barplot 預設 estimator 是 mean（要 sum 必須明確指定 estimator=sum）；③ lineplot 不適合類別資料（要連續軸）。
🎯 考古題對應 · 科二題 46 / 47 題 46：用 seaborn 比較四地區銷售比例 → 標準答案是 sns.barplot(x="variable", y="value", data=pd.melt(...), estimator=sum)（必須先 melt + 用 estimator=sum 取總額）。陷阱：① countplot 不能傳 list；② lineplot 不適合類別；③ histplot 是分布、不是比例。
題 47：NA 前 5 名遊戲長條圖 → 標準答案是 sns.barplot(x="Name", y="NA_Sales", data=df.nlargest(5, "NA_Sales"))。陷阱：① data.head(5) 是「前 5 列」不是「最大 5 名」；② lineplot 對類別 x 不適當；③ countplot 只計次不看 y。

E · sklearn LinearRegression vs statsmodels OLS

Python 有兩個主流的線性迴歸庫，用途不同必須分清：scikit-learn 適合「預測」、statsmodels 適合「推論」。考題常考兩者 API 差異。

📄 regression_compare.py · sklearn vs statsmodels（題 50 標準答案）
import pandas as pd
from sklearn.linear_model import LinearRegression
import statsmodels.api as sm

# 假設 X 是 DataFrame（含 youtube, facebook, newspaper 三特徵），y 是 sales
X = df[['youtube', 'facebook', 'newspaper']]
y = df['sales']

# ① sklearn LinearRegression — 預測導向
reg = LinearRegression().fit(X, y)  # 注意順序：fit(X, y) 不是 fit(y, X)！
print(reg.coef_)        # 3 個係數（對應 3 個特徵，不含截距）
print(reg.intercept_)   # 截距項 b
print(reg.score(X, y))  # R² 決定係數

# ② statsmodels OLS — 統計推論導向
X2 = sm.add_constant(X)             # 必須手動加截距欄（sklearn 自動加）
model_sm = sm.OLS(y, X2).fit()       # 注意：sm.OLS(y, X) 順序與 sklearn 相反！
print(model_sm.summary())
  # 輸出包含：coef / std err / t 值 / p 值 / 95% CI / R² / F-test
  # ⭐ p 值 < 0.05 的係數才具顯著性
⚠️ 兩個庫的 fit() 順序顛倒，最常踩雷 sklearn： LinearRegression().fit(X, y)  X 在前、y 在後
statsmodels： sm.OLS(y, X2).fit()  y 在前、X 在後
寫錯就跑出莫名其妙的結果 — 但不一定報錯，因為 sklearn 看到「m × 3」當 X、「m × 1」當 y 算得出來，只是參數倒過來毫無意義。
🎯 考古題對應 · 科二題 50 題 50 多選題的正確選項是 B + F：
✓ B：sklearn 寫法是 LinearRegression().fit(X, y)（X 先 y 後）
✗ A：.fit(y, X) 順序錯
✗ C：reg.coef_ 只給係數不含截距 — 截距在 reg.intercept_，所以「包括截距等 4 個係數」是錯的
✗ D：sm.OLS(X2, y) 順序錯（應該 sm.OLS(y, X2)）
✗ E：「所有係數都顯著」要看 summary() 中每個 p 值，newspaper 通常不顯著（常見廣告效益實驗結果）
✓ F：截距項係數值要從 reg.intercept_ 或 model_sm.params[0] 讀取，特定資料集（marketing.csv）截距 = 3.5561 為標準答案
📌 補充 · 何時用 sklearn vs statsmodels？ • sklearn LinearRegression：要做預測 + 部署、要與其他 ML 模型（隨機森林、XGBoost）統一 API、要做 Pipeline / CV → 標準選擇
• statsmodels OLS：要做統計推論（p 值、信賴區間、假設檢定）、要寫學術論文、要做殘差分析（殘差圖、QQ-plot、Durbin-Watson）→ 標準選擇
• 實務常見：先用 statsmodels 做探索性分析 + 確認假設，確定模型後再用 sklearn 部署上線。
